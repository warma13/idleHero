# AI 工作流指令

> 每次新会话开始时，必须遵循以下流程。

---

## 启动流程

### 第 1 步: 读取速查表

```
读取 memory/architecture.md
```

根据用户的问题/任务，在速查表中查找目标模块的位置和职责。

### 第 2 步: 定位目标文件

- **找到了** → 直接去读对应文件，开始工作
- **没找到** → 用 Explore/Grep/Glob 搜索实际文件，找到后**更新速查表**

### 第 3 步: 工作中维护速查表

每次修改代码后，检查速查表是否仍然准确:

| 变更类型 | 速查表操作 |
|---------|-----------|
| 新增文件/模块 | 添加到对应目录区 |
| 删除文件/模块 | 从速查表移除 |
| 修改模块职责 | 更新描述 |
| 修改依赖关系 | 更新依赖说明 |
| 修复已知待办 | 从"已知待办"中勾选或移除 |
| 发现新的技术债 | 添加到"已知待办" |

---

## 速查表文件

| 文件 | 用途 |
|------|------|
| `memory/architecture.md` | 项目架构、模块职责、依赖关系、已知待办 |

---

## 关键原则

1. **速查表优先** — 不要每次都从头探索整个项目
2. **信任但验证** — 速查表可能过时，读实际文件后如有出入，立即更新
3. **保持精简** — 速查表只记录"在哪里"和"做什么"，不记录具体实现细节
4. **及时更新** — 改完代码就更新，不要攒着
