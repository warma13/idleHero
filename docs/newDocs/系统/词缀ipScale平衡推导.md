# 词缀 ipScale 数值平衡推导

> 本文档记录词缀 `ipScale` 值的完整推导过程，作为后续数值调整的参考依据。

---

## 1. 背景

### 1.1 CalcAffixValue 公式

```
AffixValue = base × (1 + (IP/100 - 1) × ipScale) × roll
```

- `ipScale = 0` → 值不随 IP 增长，仅受 roll 区间影响
- `ipScale = 1.0` → 值与 IP 线性增长（IP=600 时 ipFactor=6.0）

### 1.2 问题

原始设计中只有 `atk / hp / def / hpRegen` 四个固定值词缀的 ipScale=1.0，其余 25 个词缀 ipScale=0。

后果：后期（IP=600）攻击力词缀随 IP 增长 6 倍，而元素增伤、暴击伤害等百分比词缀完全不增长，导致：
- 后期只追求 ATK 词缀，其他攻击词缀沦为"废词缀"
- A 桶词缀在后期毫无体感（每条仅贡献 ~1.7% DPS）

## 2. 伤害公式分解

### 2.1 6-Bucket D4 伤害管线

```
FinalDmg = Base × MainStat × Additive × Crit × Vulnerable × XDamage
         × DefZone × ResZone
         + Overpower
```

| 桶 | 代号 | 公式 | 词缀映射 |
|----|------|------|---------|
| Bucket 0 | Base | `ATK × SkillMultiplier` | atk |
| Bucket 1 | MainStat | `1 + INT × 0.001` | 无词缀（核心属性） |
| Bucket 2 | Additive (A) | `1 + Σ(elemDmg + vulnDmg + elite_hunter + ...)` | fireDmg, iceDmg, lightningDmg, vulnerableDmg, elite_hunter |
| Bucket 3 | Crit | `critDmg if crit, else 1.0` | crit（触发率）, critDmg（倍率） |
| Bucket 5 | XDamage (X) | `Π(1 + Xi)`，技能时含 skillDmg | skillDmg |
| Bucket 6 | Overpower | `(HP+Shield) × (1+overpowerDmg%)`, 3%触发 | overpowerDmg |

### 2.2 简化 DPS 公式

```
DPS ∝ ATK × SPD × (1 + A) × CritExp × (1 + S×u)
```

- `A` = A 桶总和（所有 additive 词缀）
- `CritExp` = `1 + critRate × (critDmg - 1)`（暴击期望乘数）
- `S` = skillDmg 总和，`u` = 技能伤害占比（约 0.6~0.7）

## 3. 基准面板估算

### 3.1 基础常量

| 常量 | 值 | 来源 |
|------|-----|------|
| baseAtk | 18 | Config.PLAYER.baseAtk |
| baseCritDmg | 1.80 | Config.PLAYER.baseCritDmg |
| baseCritRate | 0.05 | GameState init |
| baseHP | 500 | Config.PLAYER.baseHP |
| weaponAtkSpeed | 1.0 | Config.PLAYER.atkSpeed |

### 3.2 IP=100 面板（开局，chapter 1 橙装）

假设：5 件橙装 × 5 词缀 = 25 条词缀，avgRoll = avg(0.15, 0.30) = 0.225

| 统计量 | 计算 | 值 |
|--------|------|-----|
| ATK 词缀 | ~3 条, 40×1.0×0.225×3 = 27 | ATK 总 ≈ 45 |
| SPD | ~2 条, 0.03×0.225×2 = 0.014 | SPD ≈ 1.014 |
| A 桶 | ~3 条, avg 0.05×0.225×3 ≈ 0.034 | 1+A ≈ 1.034 |
| critRate | 0.05 + 2×0.04×0.225 = 0.068 | 6.8% |
| critDmg | 1.8 + 2×0.08×0.225 = 1.836 | 183.6% |
| CritExp | 1 + 0.068×0.836 = 1.057 | 1.057 |
| skillDmg | ~1 条, 0.06×0.225 = 0.014 | 1+S ≈ 1.009 |

**DPS(IP=100) ∝ 45 × 1.014 × 1.034 × 1.057 × 1.009 ≈ 49.7**

### 3.3 IP=600 面板（chapter 16-17 满配橙装，ipScale=0 旧方案）

假设：6 件 × 5 词缀 = 30 条，avgRoll = avg(0.28, 0.50) = 0.39

| 统计量 | 计算 | 值 |
|--------|------|-----|
| ATK 词缀 | ~4 条, 40×6.0×0.39×4 = 374 | ATK 总 ≈ 900 |
| SPD | ~3 条, 0.03×0.39×3 = 0.035 | SPD ≈ 1.035 |
| A 桶 | ~6 条, 0.05×0.39×6 ≈ 0.117 | 1+A ≈ 1.117 |
| critRate | 0.05 + 3×0.04×0.39 = 0.097 | 9.7% |
| critDmg | 1.8 + 3×0.08×0.39 = 1.894 | 189.4% |
| CritExp | 1 + 0.097×0.894 = 1.087 | 1.087 |
| skillDmg | ~2 条, 0.06×0.39×2 = 0.047 | 1+S ≈ 1.031 |

**DPS(IP=600, 旧) ∝ 900 × 1.035 × 1.117 × 1.087 × 1.031 ≈ 1165**

**旧方案 DPS 增长倍率：1165 / 49.7 ≈ 23.4×**

## 4. 边际 DPS 贡献分析

### 4.1 边际公式

```
ΔDPS% = ΔValue / (BucketBase + BucketTotal)
```

### 4.2 IP=600 旧方案各词缀边际贡献（单条, avgRoll=0.39）

| 词缀 | 公式层 | ΔValue | BucketTotal | ΔDPS% |
|------|--------|--------|-------------|-------|
| atk | Base(×) | 40×6×0.39 = 93.6 | 900 | **10.4%** |
| spd | SPD(×) | 0.03×0.39 = 0.012 | 1.035 | **1.15%** |
| crit | CritExp | 0.04×0.39×0.894 = 0.014 | 1.087 | **1.29%** |
| critDmg | CritExp | 0.08×0.39×0.097 = 0.003 | 1.087 | **0.28%** |
| fireDmg | A(+) | 0.05×0.39 = 0.020 | 1.117 | **1.75%** |
| vulnerableDmg | A(+) | 0.04×0.39 = 0.016 | 1.117 | **1.40%** |
| elite_hunter | A(+) | 0.25×0.39 = 0.098 | 1.117 | **8.72%**（仅Boss） |
| skillDmg | X(×) | 0.06×0.39 = 0.023 | 1.047 | **2.23%**（技能时） |
| combo_strike | Proc(×) | 0.20×0.39 = 0.078 | — | **7.8%** |
| overpowerDmg | OP | 0.05×0.39 = 0.020 | — | <0.1%（3%触发） |

**核心发现**：ATK 每条贡献 10.4%，火焰增伤仅 1.75%，差距 6 倍。后期 A 桶词缀几乎无感。

## 5. ipScale 推导

### 5.1 平衡目标

> 同一 bucket 内的词缀，边际贡献应在 2~3 倍范围内。
> 跨 bucket 差距由乘法关系天然产生，不强制拉平。

### 5.2 分层原则

| 层级 | ipScale | 适用条件 | 词缀 |
|------|---------|---------|------|
| 固定值层 | 1.0 | 固定值数值，需与怪物数值同步 | atk, hp, def, hpRegen |
| 成长层 | 0.3~0.7 | 百分比伤害/防御，需后期有体感 | 元素增伤、暴击伤害、技能伤害等 |
| 固定层 | 0 | 有硬顶/概率/经济属性 | 暴击率、攻速、连击、幸运等 |

### 5.3 各词缀推导

#### A 桶（additive）词缀

**目标**：IP=600 时 6 条 A 桶词缀总和控制在 0.3~0.6，避免膨胀导致单条无感。

| ipScale | ipFactor(IP=600) | 6 条 A 桶总量 | 评估 |
|---------|:-----------------:|:-------------:|------|
| 0 | 1.0 | 0.117 | 过低，无体感 |
| 0.3 | 2.5 | 0.293 | 偏低 |
| 0.4 | 3.0 | 0.351 | ✓ 合适 |
| 0.5 | 3.5 | 0.410 | ✓ 合适 |
| 0.6 | 4.0 | 0.468 | ✓ 可接受 |
| 0.8 | 5.0 | 0.585 | 接近上限 |

**推荐**：

| 词缀 | ipScale | 理由 |
|------|---------|------|
| fireDmg / iceDmg / lightningDmg | **0.5** | 元素增伤是核心 build 构建要素 |
| vulnerableDmg | **0.4** | base 较低(0.04)，0.4 保持相对体感 |
| elite_hunter | **0.3** | base 很高(0.25)，仅对 Boss 生效，避免膨胀 |

#### 暴击桶

- `crit`（暴击率）：有 100% 硬顶，**ipScale = 0**
- `critDmg`（暴击伤害）：无上限，但与 critRate 联动

目标：IP=600 时 critDmg 从 1.89 涨到 ~2.2（+120% 暴击伤害）

| ipScale | ipFactor | 3 条 critDmg 增量 | critDmg 总 |
|---------|:--------:|:-----------------:|:----------:|
| 0 | 1.0 | 0.094 | 1.894 |
| 0.5 | 3.5 | 0.328 | 2.128 |
| 0.7 | 4.5 | 0.421 | 2.221 |
| 1.0 | 6.0 | 0.562 | 2.362 |

**推荐 critDmg ipScale = 0.7**（critDmg 总 ≈ 2.22，暴击伤害 +122%）

#### X-damage 桶

skillDmg 是独立乘区，不会被同桶稀释，不能给太高。

| ipScale | ipFactor | 2 条 skillDmg 总 | 技能 DPS 增幅 |
|---------|:--------:|:----------------:|:------------:|
| 0 | 1.0 | 4.7% | +3.1% |
| 0.3 | 2.5 | 11.7% | +7.8% |
| 0.5 | 3.5 | 16.4% | +10.9% |

**推荐 skillDmg ipScale = 0.3**（保守增长，避免技能流碾压普攻流）

#### 不增长类

| 词缀 | ipScale | 理由 |
|------|---------|------|
| spd | 0 | 有双 cap（各 100%），影响游戏节奏 |
| crit | 0 | 硬顶 100% |
| combo_strike | 0 | 概率翻倍，类似暴击率 |
| crit_surge | 0 | 暴击时固定倍率伤害 |
| overpowerDmg | 0 | 仅 3% 触发率 |
| luck / greed / scholar / lucky_star | 0 | 经济属性 |
| skillCdReduce | 0 | 渐近 cap（75%） |
| hpPct / lifeSteal / shldPct | 0 | 百分比天然随基础值增长 |
| 三抗性 | 0 | 有 75% 抗性上限 |
| last_stand / kill_heal | 0 | 概率/百分比触发 |
