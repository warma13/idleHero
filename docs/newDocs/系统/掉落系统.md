# 掉落系统

## 概述

掉落系统采用**数据驱动架构**，通过 `DropManager` 中的声明式 `DROP_RULES` 掉落规则表管理所有掉落逻辑。敌人死亡后由 `StageManager.OnEnemyKilled()` 统一触发，经由 `DropManager.ProcessDrops()` 遍历规则表生成掉落物，最终由 `Loot` 模块处理物理散落和磁吸拾取。

---

## 核心流程

```
敌人死亡
  │
  ▼
StageManager.OnEnemyKilled(bs, enemy)      ← battle/StageManager.lua:38
  │
  ├─ 更新击杀计数 (killCount, totalKills)
  │
  ▼
DropManager.ProcessDrops(bs, enemy, mode)   ← battle/DropManager.lua:177
  │
  ├─ 构建上下文 ctx = { bs, enemy, luck, chapter, stageIdx, mode }
  │
  ├─ 遍历 DROP_RULES:
  │     对每条规则 → condition(ctx) ? execute(ctx) : 跳过
  │
  ▼
Loot.Spawn(loots, x, y, type, value, color, extra)  ← battle/Loot.lua:18
  │
  ├─ 生成随机散射方向和速度
  ├─ 插入 bs.loots 列表
  │
  ▼
Loot.Update(dt, loots, playerBattle, pickupRadius)   ← 每帧调用
  │
  ├─ 散落物理模拟 (摩擦衰减)
  ├─ 磁吸判定 (距离 / 延迟吸收)
  ├─ 拾取: Loot.Collect(loot)
  └─ 地面装备上限管理
```

## 掉落物类型

| 类型 | 标识 | 说明 | value 含义 | 拾取效果 |
|------|------|------|-----------|---------|
| 经验 | `"exp"` | 绿色光球 | 经验值数量 | `GameState.AddExp(value)` |
| 金币 | `"gold"` | 4帧金币动画 | 金币数量 | `GameState.AddGold(value)` |
| 装备 | `"equip"` | 装备图标 | 装备对象 table | `GameState.AddToInventory(value)` |
| 魂晶 | `"soulCrystal"` | 紫色晶体 4帧动画 | 魂晶数量 | `GameState.AddSoulCrystal(value)` |
| 背包物品 | `"bagItem"` | 各类消耗品 | 物品数量 | `GameState.AddBagItem(itemId, value)` |

### 背包物品子类

目前掉落规则不直接产出背包物品，但 `bagItem` 类型仍可通过 `DropManager.AddRule()` 扩展使用。

## 掉落模板 (DROP_TEMPLATES)

**位置**: `Config.lua:150-161`

每种怪物通过 `dropTemplate` 字段引用掉落模板，模板定义了金币范围和掉率：

| 模板 | 金币范围 | 金币概率 | 装备概率 | 适用对象 |
|------|---------|---------|---------|---------|
| `common` | 2~7 | 30% | 12% | 普通小怪 |
| `elite` | 5~12 | 50% | 20% | 精英怪 |
| `miniboss` | 8~18 | 100% | 50% | 小Boss (每章第5关) |
| `boss` | 20~40 | 100% | 100% | 大Boss (每章第10关) |
| `summon` | 0~1 | 10% | 0% | 召唤物 / 分裂体 |

### 模板展开流程

```
MonsterTemplates.lua                Spawner.lua:173-177
怪物行为定义                         生成敌人实例时展开模板
────────────────                    ────────────────────
dropTemplate = "common"    →    DROP_TEMPLATES["common"]
                                   goldMin = goldDrop[1]
                                   goldMax = goldDrop[2]
                                   goldChance = 0.30
                                   equipChance = 0.12
```

金币掉落量直接使用模板定义的基础值，不做缩放。

## 掉落规则表 (DROP_RULES)

**位置**: `battle/DropManager.lua:32-155`

共 4 条声明式规则，按顺序逐条判定，互不排斥（一个敌人可同时触发多条规则）。

### 规则 1: 经验

```
条件: 特殊模式未跳过经验掉落
数量: enemy.expDrop (来自 MonsterTemplates)
加成: 博学词缀 → expAmount × (1 + scholarVal)
```

### 规则 2: 金币

```
条件: math.random() < enemy.goldChance
数量: random(goldMin, goldMax)
加成: 贪婪词缀 → baseGold × (1 + greedVal)
位置: 敌人坐标 ±15 像素随机偏移
```

### 规则 3: 装备

```
条件: math.random() < (enemy.equipChance + luck × 0.5)
生成: GameState.GenerateEquip(wave, isBoss)
附带: extra = { slotId, name, setId }
颜色: equip.qualityColor (品质色)
```

### 规则 4: 魂晶 (仅Boss)

```
条件: enemy.isBoss == true
数量: SOUL_CRYSTAL.dropPerBoss (固定值，默认 1)
颜色: {160, 80, 255} (紫色)
```

## 装备生成 (GenerateEquip)

**位置**: `state/equip/ItemFactory.lua:140`

### 品质权重

| 品质 | 名称 | 基础权重 | 词缀数 | 颜色 |
|------|------|---------|--------|------|
| 1 | 白色 | 50 | 0 | `{200, 200, 200}` |
| 2 | 绿色 | 30 | 1 | `{100, 220, 100}` |
| 3 | 蓝色 | 15 | 2 | `{80, 140, 255}` |
| 4 | 紫色 | 4 | 3 | `{180, 80, 220}` |
| 5 | 橙色 | 1 | 4 | `{255, 165, 0}` |

### 品质计算公式

```lua
for 每个品质 i:
    w = dropWeight

    -- 低品质(白/绿): 有幸运值时权重降低
    if i <= 2 then
        w = w × max(0.3, 1 - luck)

    -- 高品质(蓝+): 幸运值提升权重
    else
        w = w × (1 + luck × (i - 2))

        -- 幸运星词缀额外加成
        if luckyStarVal > 0 then
            w = w × (1 + luckyStarVal)
        end
    end

    -- 非Boss小怪的品质压制倍率
    if not isBoss then
        mobMul = { 1, 1, 0.054, 0.02, 0.008 }
        w = w × mobMul[i]
    end
```

**小怪 vs Boss 的品质差异**：小怪蓝色装备权重仅 Boss 的 5.4%，紫色 2%，橙色 0.8%。

### 装备数据结构

```lua
{
    slot          -- 部位 ID
    slotName      -- 部位名称
    qualityIdx    -- 品质索引 (1~5)
    qualityName   -- 品质名称
    qualityColor  -- 品质颜色 RGB table
    itemPower     -- 装备战力值
    affixes       -- 词缀列表
    setId         -- 套装 ID (nil = 非套装; v12: 普通掉落不产出套装)
    upgradeLv     -- 强化等级
    mainStatId    -- 主属性 ID
    mainStatBase  -- 主属性基础值
    mainStatValue -- 主属性当前值
    sockets       -- 宝石孔 (仅橙色)
    gems          -- 已镶嵌宝石
    name          -- 装备名称
}
```

## 掉落物物理与拾取

**位置**: `battle/Loot.lua`

### 散落参数

| 参数 | 值 | 说明 |
|------|-----|------|
| `SCATTER_SPEED_MIN` | 100 | 散射最小初速度 (px/s) |
| `SCATTER_SPEED_MAX` | 200 | 散射最大初速度 (px/s) |
| `SCATTER_FRICTION` | 2.0 | 摩擦衰减系数 |
| `ABSORB_DELAY` | 2.0 | 掉落后 2 秒自动全屏磁吸 |
| `MAX_GROUND_EQUIPS` | 100 | 地面装备最大堆积数 |

### 掉落物数据结构

```lua
{
    x, y,           -- 当前位置
    vx, vy,         -- 散射速度 (摩擦衰减)
    type,           -- 掉落类型 ("exp" / "gold" / "equip" / ...)
    value,          -- 值或对象
    color,          -- 颜色 RGB table
    extra,          -- 额外数据 { slotId, name, setId, itemId }
    life,           -- 剩余生存时间 (默认 15 秒)
    age,            -- 已存在时间
    attracted,      -- 是否正在被磁吸
    scattering,     -- 是否正在散落阶段
}
```

### 拾取流程

```
每帧 Loot.Update():
  │
  ├─ 散落阶段: 物理模拟 (vx/vy 按摩擦衰减, 速度 < 5 停止)
  │
  ├─ 磁吸触发条件 (散落结束后):
  │   ├─ 1) 距离 < pickupRadius         → 进入拾取范围
  │   ├─ 2) age > ABSORB_DELAY (2秒)    → 延迟自动吸收
  │   └─ 3) 装备符合自动分解条件         → 背包满也吸收
  │
  ├─ 磁吸移动: speed = 300 + (200 - min(dist, 200)) × 3
  │   (越近越快, 最大 900 px/s)
  │
  ├─ dist < 12 时调用 Loot.Collect(loot)
  │   ├─ 成功 → 移除掉落物
  │   └─ 失败 (背包满) → 取消吸引, 落回地面
  │
  └─ 背包满时: EnforceGroundEquipCap() 控制上限
```

### 地面装备上限

背包满时，地面装备超过 100 件触发清理：
- **排序依据**: 品质升序 (低品质优先清理)，同品质按掉落时间升序 (老的先清理)
- **清理数量**: 超出部分全部移除

### 自动分解机制

背包满时装备默认不被磁吸（留在地面）。但如果玩家配置了自动分解规则，则符合条件的装备仍会被吸收：

```lua
GameState.autoDecompConfig = { 0, 0, 0, 0, 0 }  -- 每品质的分解模式

-- 从高品质向低品质扫描，找到第一个激活的阈值
-- aLv: 品质阈值 (1~5)
-- aMode: 1 = 保留该品质套装, 2 = 全分解含套装

-- 吸收条件: 装备品质 <= aLv 且 (aMode==1 时非套装, 或 aMode==2)
```

### 装备生存续命

背包满时装备类掉落物 `life` 耗尽不消失，自动续命 10 秒等待背包腾出空间。

## 词缀对掉落的影响

| 词缀 | ID | 影响 | 公式 |
|------|-----|------|------|
| 博学 | `scholar` | 经验掉落量 +N% | `exp × (1 + scholarVal)` |
| 贪婪 | `greed` | 金币掉落量 +N% | `gold × (1 + greedVal)` |
| 幸运星 | `lucky_star` | 蓝色以上装备权重 +N% | `weight × (1 + luckyStarVal)` (品质 >= 3) |
| — | `luck` (属性) | 装备掉率 / 高品质权重 | 掉率: `equipChance + luck × 0.5`; 白绿权重 × `max(0.3, 1-luck)`, 蓝+权重 × `(1 + luck×(i-2))` |

## 扩展接口

`DropManager` 提供动态注册/移除掉落规则的 API，供 GameMode 或扩展系统使用：

```lua
-- 添加自定义掉落规则
DropManager.AddRule({
    id = "my_custom_drop",
    condition = function(ctx)
        return ctx.enemy.isBoss and ctx.chapter >= 5
    end,
    execute = function(ctx)
        Loot.Spawn(ctx.bs.loots, ctx.enemy.x, ctx.enemy.y,
            "bagItem", 1, { 255, 255, 0 }, { itemId = "my_item" })
    end,
})

-- 移除指定规则 (如特殊模式下禁用经验掉落)
DropManager.RemoveRule("exp")
```
