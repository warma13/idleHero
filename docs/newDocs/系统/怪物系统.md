# 怪物系统

## 1. 系统概览

怪物系统采用 **三层正交架构**，通过组合而非穷举来定义怪物：

```
怪物家族 (MonsterFamilies)    ← 外观、抗性、种族档位
    ×
行为模板 (MonsterTemplates)   ← 基础属性、能力标签、抗性计算
    ×
Boss 原型 (BossArchetypes)    ← 阶段技能、元素注入
    ↓
无限章节生成器 (StageConfig)  ← 公式化编排关卡波次
    ↓
Spawner / EnemySystem         ← 运行时实例化
```

**核心设计思想**：8 个家族 × 7 种行为 = 56 种普通怪物，7 个 Boss 原型 × 6+ 种元素 = 168+ 种 Boss，章节无限延伸。

**代码结构**（`scripts/`）：

| 模块 | 文件 | 职责 |
|------|------|------|
| 怪物家族 | `MonsterFamilies.lua` | 8 家族定义、种族档位、成员贴图、家族解析 |
| 行为模板 | `MonsterTemplates.lua` | 7 行为模板、12 抗性模板、能力标签、章节主题、组装函数 |
| Boss 原型 | `BossArchetypes.lua` | 7 Boss 原型、元素风味注入、阶段/技能定义 |
| 关卡配置 | `StageConfig.lua` | 无限章节生成器、10 关编排模板、怪物解析接口 |

**关联模块**：

| 模块 | 文件 | 关系 |
|------|------|------|
| 生成队列 | `battle/Spawner.lua` | 调用 `StageConfig.GetStage()` + `ResolveMonster()` 构建出怪队列 |
| 敌人实例 | `battle/EnemySystem.lua` | 根据怪物定义实例化敌人实体（HP/ATK/速度/标签） |
| 动态等级 | `battle/DynamicLevel.lua` | 运行时按等级缩放 HP/ATK（种族基准值 × scaleMul） |
| 关卡选择 | `ui/StageSelect.lua` | 展示章节/关卡选择 UI，调用 `GetStage()`/`GetStageCount()` |

## 2. 种族档位系统

每只怪物被分配一个种族档位（S/A/B/C/D），决定其基准 HP/ATK/DEF。种族档位 **取代** 行为模板中的基础属性，确保同一行为模板在不同种族下有差异化的生存力。

### 2.1 五档定义

| 档位 | 基准 HP | 基准 ATK | 基准 DEF | 典型种族 |
|------|---------|---------|---------|---------|
| **S** | 55 | 8 | 0 | 飞行小型：蝙蝠、幽灵、精灵、微粒 |
| **A** | 85 | 12 | 5 | 远程单位：术士、弓手、祭司、编织者 |
| **B** | 110 | 13 | 15 | 标准近战：骸骨、劫匪、鱼人、蜥蜴 |
| **C** | 240 | 18 | 35 | 重型近战：骑士、蟹、蝎、守卫、傀儡 |
| **D** | 400 | 24 | 50 | 超重型：泰坦、巨像、树人、行为兽 |


### 3.3 成员属性覆盖

每个成员可携带个性化覆盖：

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `name` | string | 显示名称 | "骸骨鼠" |
| `image` | string | 贴图路径 | "Textures/mobs/grave_rat.png" |
| `resistId` | string\|nil | 覆盖家族默认抗性 | tank 常用 "phys_armor"，glass/exploder 常用 "all_low" |
| `color` | number[]\|nil | 覆盖色调 | nil 则使用家族 colorBase |
| `tags` | table\|nil | 额外能力标签 | `{ someTag = 3 }` |
| `overrides` | table\|nil | 属性覆盖 | `{ speed = 80 }` |
| `raceTier` | string\|nil | 成员级种族档位覆盖 | "C" |

### 3.4 解析流程

`MonsterFamilies.Resolve(familyId, behaviorId, chapter, tagLevels, monsterId)`:

```
1. 查找家族 + 成员定义
2. 确定抗性模板: 成员级 resistId > 家族 resistProfile
3. 合并能力标签: 行为默认标签 + 章节可选标签 + 成员额外标签
4. 调用 MonsterTemplates.Assemble() 组装基础定义
5. 种族档位覆盖 hp/atk/def（成员级 > MONSTER_RACE 表）
6. 记录 familyId + behaviorId 元数据
```

`MonsterFamilies.ResolveById(compositeId, chapter, tagLevels)`:
- 解析 `"familyId_behaviorId"` 格式（如 `"undead_swarm"`）
- 内部调用 `Resolve()`

## 5. 能力标签系统

### 5.1 标签等级参数

每个标签有 1~5 级，等级决定具体数值。以下是所有标签的详细参数：

#### 减益类标签

| 标签 | 等级 | 关键参数 |
|------|------|---------|
| **slowOnHit** | 1~5 | 减速 20%→50%，持续 1.5s→3.0s |
| **sporeCloud** | 1~5 | 降低攻速 15%→35%，持续 4.0s→5.0s |
| **venomStack** | 1~5 | 每层 1.5%→4.0% 伤害，上限 5~8 层 |
| **corrosion** | 1 | 降低防御 8%/层，最多 5 层，持续 8s |
| **inkBlind** | 1 | 降低攻击 25%，持续 4s |
| **sandStorm** | 1 | 降低暴击 20%，持续 5s |

#### 攻击增强类标签

| 标签 | 等级 | 关键参数 |
|------|------|---------|
| **defPierce** | 1~5 | 穿透 25%→50% 防御 |
| **firstStrikeMul** | 1~4 | 首击伤害 ×2.0→×2.8 |
| **packBonus** | 1~5 | 集群加成 25%→45%，阈值 3~5 只 |
| **chargeUp** | 1~2 | 蓄力 5~8 层，伤害 ×2.0~2.5，2级有 AOE |
| **chainLightning** | 1 | 弹射 2 次，伤害 ×0.5，范围 80 |

#### 防御/续航类标签

| 标签 | 等级 | 关键参数 |
|------|------|---------|
| **lifesteal** | 1~4 | 吸血 12%→20% |
| **hpRegen** | 1~3 | 回复 2%→3% HP，间隔 5~6s |
| **healAura** | 1~5 | 治疗 4%→8% HP，间隔 7s，半径 90~120 |

#### 死亡触发类标签

| 标签 | 等级 | 关键参数 |
|------|------|---------|
| **deathExplode** | 1~5 | 爆炸伤害 ×0.6→×1.5，半径 35→58 |
| **splitOnDeath** | 1 | 分裂 2 个子体，40% HP/ATK |

#### 布尔标签

| 标签 | 说明 |
|------|------|
| **antiHeal** | 禁止被治疗 |
| **isRanged** | 远程攻击 |

### 5.2 标签合并逻辑

```
最终标签 = 行为模板默认标签
         + 章节可选标签（满足引入章节门槛 && tagLevels 中声明）
         + 成员级额外标签
```

### 6.3 抗性示例

```

ice 抗性      = 5  × 0.85 / 100 = 0.0425 (4.25% 冰霜减伤)
physical 抗性 = 15 × 0.85 / 100 = 0.1275 (12.75% 物理减伤)
```

### 7.4 元素风味注入

同一原型在不同元素下产生完全不同的战斗体验：

| 元素 | 命中效果 (onHit) | 衰减目标 | 护盾反应 | 弱点元素 |
|------|-----------------|---------|---------|---------|
| **ice** | 减速 10%，1.5s | 移动速度 | 融化 (melt) | fire |
| **poison** | 叠加毒素减益 | 防御 | 净化 (purify) | fire |
| **fire** | 施加灼烧减益 | 攻速 | 淬灭 (quench) | water |
| **arcane** | 沉默 1.5s | 暴击 | 驱散 (dispel) | holy |
| **water** | 腐蚀 3%/层 | 治疗降低 | 蒸发 (evaporate) | fire |
| **holy** | 致盲 2.0s | 伤害降低 | 审判 (judgment) | arcane |

**注入规则**：
- 攻击技能（barrage/breath）→ 注入 onHit 效果
- 衰减技能（CTL_decay）→ 注入衰减目标属性
- 护盾技能（DEF_shield）→ 注入反应类型 + 弱点元素（弱点伤害 ×2.5）
- 领域技能（CTL_field）→ 注入 onHit 作为领域效果
- 漩涡技能（CTL_vortex）→ 注入 onHit 作为核心效果

### 7.5 Boss 命名规则

## 8. 无限章节生成器

### 8.1 设计原理

章节不再手写，而是通过公式动态生成。`StageConfig.CHAPTERS` 使用 metatable 代理：

```lua
StageConfig.CHAPTERS = setmetatable({}, {
    __index = function(_, k)
        -- 惰性生成第 k 章配置
        return GenerateChapter(k)
    end,
    __len = function()
        return 999  -- 逻辑安全上限
    end,
})
```

### 8.2 GenerateChapter(n) 公式

每个章节由以下公式决定：

| 维度 | 公式 | 说明 |
|------|------|------|
| **视觉主题** | `CHAPTER_VISUALS[((n-1) % 17) + 1]` | 17 个主题循环 |
| **主家族** | `FAMILY_IDS[((n-1) % 8) + 1]` | 8 家族轮换 |
| **副家族** | `FAMILY_IDS[(n % 8) + 1]` | 相邻家族配对 |
| **中 Boss 原型** | `MID_BOSS_ARCHETYPES[((n-1) % 4) + 1]` | 4 原型轮换 |
| **终 Boss 原型** | `FINAL_BOSS_ARCHETYPES[((n-1) % 3) + 1]` | 3 原型轮换 |
| **Boss 元素** | 主家族对应元素 | 与主家族绑定 |

**家族 → 元素映射**：

| 家族 | 元素 |
|------|------|
| undead | arcane |
| beast | physical |
| elemental_fire | fire |
| elemental_ice | ice |
| elemental_poison | poison |
| arcane | arcane |
| divine | holy |
| aquatic | water |

### 8.3 章节配置输出格式

```lua
{
    id = n,
    name = "灰烬荒原",        -- 视觉主题名
    desc = "焰息城外的第一步",  -- 视觉主题描述
    families = { "undead", "beast" },  -- 主/副家族
```

## 9. 关卡编排模板 (STAGE_TEMPLATES)

每章固定 10 关，使用 10 个编排模板定义波次组成：

| 关卡 | 模板 ID | 波次 | 设计意图 |
|------|---------|------|---------|
| 1 | `swarm_only` | 2 波 | 纯蜂群入门，熟悉家族外观 |
| 2 | `intro_tank` | 2 波 | 引入肉盾，需要集火 |
| 3 | `glass_rush` | 2 波 | 脆皮海，高速高伤低血 |
| 4 | `intro_bruiser` | 2 波 | 引入精英，中等难度 |
| 5 | `mid_boss` | 2 波 | **中 Boss**，第 2 波出 Boss + 小怪 |
| 6 | `mixed_intro` | 2 波 | 引入副家族，debuffer + caster 混合 |
| 7 | `slow_hell` | 2 波 | 减速地狱，debuffer + tank 压制 |
| 8 | `triple_wave` | 3 波 | 三波大混战，glass + bruiser + exploder |
| 9 | `high_density` | 3 波 | 高密度三波，全角色混合 |
| 10 | `final_boss` | 2 波 | **终 Boss**，第 2 波出 Boss + 混合小怪 |

### 9.1 波次角色分配

编排模板使用角色引用（`role`）而非具体怪物 ID：

```lua
{ role = "swarm", count = 25, family = "primary" }
-- → 运行时解析为: primaryFamily .. "_swarm" → "undead_swarm"
```

- `family = "primary"` → 使用章节主家族
- `family = "secondary"` → 使用章节副家族
- Boss 角色无 family 字段，直接从章节 boss 配置解析

### 9.2 从模板到关卡的转换

`generateStages(chapterCfg, chapter)` 函数：

```
1. 遍历 10 个 STAGE_TEMPLATES
2. 每个角色引用 → 组合为 "familyId_behaviorId" 格式的怪物 ID
3. 首次出现的怪物 ID → 调用 MonsterFamilies.Resolve() 生成定义
4. 注册到 StageConfig.MONSTERS 表（惰性缓存）
5. Boss 角色 → 调用 BossArchetypes.Resolve() 生成，ID 含 "_ch{n}" 后缀
6. 输出与手写 stages 格式完全一致的关卡配置
```

## 10. 怪物解析接口 (ResolveMonster)

`StageConfig.ResolveMonster(monsterId, chapter, tagLevels)` 是 Spawner 的统一入口：

```

   ↓ 解析失败
4. 返回 nil
```

## 11. 双轨兼容设计

系统支持两种章节配置格式，通过 `GetStage()` 自动适配：

| 格式 | 字段 | 来源 | 说明 |
|------|------|------|------|
| 手写关卡 | `ch.stages` | 旧版手写 | 直接返回 `ch.stages[stage]` |
| 公式生成 | `ch.families` | GenerateChapter | 调用 `generateStages()` 自动编排 |

```lua
function StageConfig.GetStage(chapter, stage)
    local ch = StageConfig.CHAPTERS[chapter]
    if ch.stages then
        return ch.stages[stage]        -- 手写路径
    elseif ch.families then
        local generated = getGeneratedStages(ch, chapter)
        return generated[stage]        -- 自动编排路径
    end
end
```

## 13. 数值缩放管线

完整的数值流从定义到实战：

```
种族基准值 (hp/atk/def)           ← MonsterFamilies.RACE_TIERS
    × scaleMul (章节+关卡难度)    ← StageConfig.CalcScaleMul()
    = 运行时战斗属性               ← Spawner / DynamicLevel

行为模板 (speed/atkInterval)       ← MonsterTemplates.Behaviors
    不缩放，固定值                  ← 直接使用

抗性模板 × 章节系数 / 100         ← MonsterTemplates.CalcResist()
    = 最终抗性百分比               ← 应用于伤害计算

Boss: BossBase × 原型倍率         ← BossArchetypes.Resolve()
    × scaleMul                    ← Spawner 运行时缩放
```

### 13.1 scaleMul 完整计算公式

```
scaleMul = difficultyRatio × tierMul × powerMul
```

三个因子详解：

**① tierMul — 章节装备等级倍率**

来自 `ConfigCalc.GetChapterTier(chapter)`：

```
tierMul = 1.0 + 99 × ln(chapter) / ln(100)
```

| 章节 | tierMul |
|------|---------|
| 1 | 1.0 |
| 5 | 35.6 |
| 10 | 50.5 |
| 17 | 61.9 |

**② difficultyRatio — 关卡内指数递增**

```lua
chGrowth = 1 + 0.06 × (chapter - 1)      -- 每章 +6%
baseRatio_s1 = 1.15 × chGrowth            -- 第 1 关基础难度
STAGE_RAMP = 5.5714                        -- 章末 / 章首 = 5.57 倍
t = (stageIdx - 1) / (stageCount - 1)     -- 0.0 ~ 1.0
difficultyRatio = baseRatio_s1 × STAGE_RAMP^t
```

效果：同一章节内，第 10 关怪物属性约为第 1 关的 **5.57 倍**。

**③ powerMul — 玩家 DPS 增长补偿 (v4: 技能百分比体系)**

公式化计算（`getPlayerPowerMul(chapter)`），无需手调表：

```lua
-- 玩家ATK增长
ip = CalcBaseIP(chapter)                -- 100 + 825*ln(ch)/ln(100)
atkRatio = (18 + 70*(ip/100)) / 88     -- 相对ch1的ATK倍率

-- 技能解锁带来的DPS跳跃
skillFactor = 1.0    -- ch1~3: fire_bolt (27%)
            | 3.56   -- ch4~8: fireball (96%) / fire_bolt (27%)
            | 11.85  -- ch9+:  fire_storm (320%) / fire_bolt (27%)

-- 最终: 补偿ATK增长与tierMul之间的差距
powerMul = atkRatio × skillFactor / tierMul × K   -- K=4.0
```

设计原理：`tierMul` 增长远快于玩家 ATK（99 倍系数 vs ~4 倍系数），
`powerMul` 将两者的差距补回来，使最终击杀次数只随 `difficultyRatio` 缓慢增长。

| 章节 | ATK | ATK倍率 | 主力技能 | skillFactor | powerMul | 击杀次数(swarm) |
|------|-----|--------|---------|-------------|----------|---------------|
| 1 | 88 | 1.0x | fire_bolt 27% | 1.0 | 4.0 | ~8 |
| 3 | 226 | 2.6x | fire_bolt 27% | 1.0 | 0.4 | ~9 |
| 5 | 290 | 3.3x | fireball 96% | 3.56 | 1.3 | ~10 |
| 10 | 377 | 4.3x | fire_storm 320% | 11.85 | 4.0 | ~12 |
| 17 | 443 | 5.0x | fire_storm 320% | 11.85 | 3.8 | ~16 |

**scaleMul 示例**（第 5 章第 1 关 vs 第 5 章第 10 关）：

```
ch5-s1:  diffRatio(1.43) × tierMul(35.6) × powerMul(1.3) ≈ 66
ch5-s10: diffRatio(7.94) × tierMul(35.6) × powerMul(1.3) ≈ 368
swarm(hp=40): s1→ 2,640 HP,  s10→ 14,720 HP
fireball(96% × 290 ATK = 278/hit): s1→10次, s10→53次击杀
```

### 18.1 运行时实体结构

Spawner 生成后，每个怪物实体包含以下运行时字段：

```lua
enemy = {
    -- ── 基础属性 ──
    x, y,                    -- 位置
    hp, maxHp,               -- 生命值
    atk,                     -- 攻击力
    speed,                   -- 移动速度

    element,                 -- 元素类型
    radius,                  -- 碰撞半径
    color, image, name,      -- 视觉
    isBoss, dead,            -- 标记

    -- ── 战斗计时 ──
    atkTimer, atkCd,         -- 攻击冷却
    atkRange,                -- 攻击距离 (默认 35)
    weight,                  -- 击退抗性

    -- ── 位移状态 ──
    knockbackVx, knockbackVy,-- 击退速度 (指数衰减)

    -- ── 控制状态 ──
    stunTimer,               -- 眩晕剩余时间
    slowTimer, slowFactor,   -- 减速
    frozenTimer, isFrozen,   -- 冻结
    frostbite,               -- 冰霜累计 (碎冰机制)

    -- ── Debuff ──
    defReduceRate, defReduceTimer,       -- DEF 降低
    elemWeakenRate, elemWeakenTimer,      -- 元素易伤
    vulnerableTimer, isVulnerable,       -- 脆弱
    reactionDot, igniteDot,              -- 持续伤害

    -- ── 能力标签展开后的字段 ──
    packBonus, packThreshold, _packActive,  -- 狼群
    hpRegenPct, hpRegenInterval,            -- 回血
    healAura,                               -- 治疗光环
    chargeUp, chargeUpStacks,               -- 充能
    deathExplode,                           -- 死亡爆炸
    splitOnDeath,                           -- 死亡分裂
    lifesteal, defPierce, firstStrikeMul,   -- 攻击增强
    slowOnHit, antiHeal, isRanged,          -- 攻击附加
    venomStack, sporeCloud,                 -- 特殊

    -- ── 模板/家族元数据 ──
    templateId,              -- 怪物模板 ID
    familyId,                -- 家族 ID
    behaviorId,              -- 行为 ID
    familyType,              -- 家族机制类型
    eliteRank,               -- "elite" / "champion" / nil
    affixes,                 -- 精英词缀列表

    -- ── Boss 专用 ──
    phases,                  -- 阶段配置
    _phaseIdx,               -- 当前阶段索引
    barrage, dragonBreath,   -- Boss 技能
    summon, iceArmor,        -- Boss 能力
    enraged,                 -- 狂暴状态
}
```

### 18.2 AI 行为

**近战怪物**（默认）：
```
if 距离 > atkRange then
    向玩家移动 (速度受 slowFactor 影响)
else
    原地攻击 (按 atkCd 间隔)
```

**远程怪物** (`isRanged = true`)：
```
保持距离 = 120
if 距离 > 保持距离 + 20 then 靠近
elseif 距离 < 保持距离 - 30 then 后退
else 射击
```

### 18.3 攻击伤害流程

`EnemyAttackPlayer(enemy)` 完整链路：

```
1. 原始伤害 rawDmg = atk
2. 增伤修正:
   ├─ 狼群增伤: ×(1 + packBonus)     [场上同模板 ≥ packThreshold]
   ├─ 首击倍率: ×firstStrikeMul      [第一次攻击]
   └─ 充能攻击: ×chargeUp.dmgMul     [叠满层数时]
3. 穿甲: 有效DEF = playerDEF × (1 - defPierce)
4. DEF减免: ×(1 - DEF/(DEF + 200))
5. 元素抗性: ×ResistMul(resist)
6. 闪避判定
7. 扣血 + 飘字
8. 附加效果:
   ├─ slowOnHit     → 减速
   ├─ antiHeal      → 降低治疗效果
   ├─ lifesteal     → 回血 (伤害 × 吸血率)
   ├─ corrosion     → 叠层降 DEF
   ├─ inkBlind      → 降低攻击力 25%
   ├─ venomStack    → 叠毒层 DoT
   ├─ burnStack     → 灼烧 DoT + 攻速降低
   ├─ sandStorm     → 降暴击率 20%
   ├─ chainLightning→ 弹射 (对其他目标无效，仅主目标)
   ├─ scorchOnHit   → 焚灼
   └─ drenchStack   → 浸蚀
```

### 18.4 状态效果处理

`UpdateEnemyDebuffs(dt)` 每帧衰减：

| 状态 | 效果 | 衰减方式 |
|------|------|---------|
| 击退 | 位移，指数衰减速度 | 每帧 ×衰减系数 |
| 眩晕 | 跳过移动和攻击 | `stunTimer -= dt` |
| 减速 | 移动速度 × slowFactor | `slowTimer -= dt` |
| 冻结 | 强减速 + 伤害累计（碎冰） | `frozenTimer -= dt` |
| 脆弱 | 受伤增加 | `vulnerableTimer -= dt` |
| DEF降低 | 减少防御值 | `defReduceTimer -= dt` |
| 元素易伤 | 增加元素伤害 | `elemWeakenTimer -= dt` |
| 持续伤害 | 每 tick 扣血 | `reactionDot` / `igniteDot` |

### 18.5 死亡处理

`EnemySystem.OnEnemyDeath(bs, enemy)` 处理：

1. **死亡分裂** (`splitOnDeath`)：产生 N 个子怪物（40% HP/ATK）
2. **孢子云** (`sporeCloud`)：降低玩家攻速
3. **死亡爆炸** (`deathExplode`)：AOE 伤害

## 19. 伤害公式

### 19.1 玩家对怪物

```
有效DEF = 怪物DEF × (1 - defReduceRate)
defMul = 1 - 有效DEF / (有效DEF + ENEMY_DEF_K)    -- K = 100
```

### 19.2 怪物对玩家

```
有效DEF = 玩家DEF × (1 - defPierce)
defMul = 1 - 有效DEF / (有效DEF + PLAYER.defK)     -- K = 200
```

### 19.3 元素抗性曲线

`Config.ResistMul(resist)` 三段曲线：

| 抗性区间 | 公式 | 说明 |
|---------|------|------|
| resist < 0 | `1 - resist/2` | 负抗性 = 易伤（受伤增加） |
| 0 ≤ resist < 0.75 | `1 - resist` | 线性减免 |
| resist ≥ 0.75 | `1 / (1 + resist/4)` | 收益递减（防止 100% 免伤） |

示例：抗性 0.5 → 受伤 ×0.5 (减半)；抗性 -0.2 → 受伤 ×1.1 (增伤 10%)。

## 20. 生成系统 (Spawner)

### 20.1 生成流程

```
Spawner.BuildQueue()
  ├─ 检查 GameMode 适配器 (噩梦地牢/深渊等特殊模式)
  ├─ 或调用 StageConfig.GetStage() 获取关卡配置
  ├─ 遍历波次的怪物列表
  ├─ 每个怪物 → StageConfig.ResolveMonster() 解析模板
  ├─ 应用 scaleMul 缩放 HP/ATK
  ├─ 应用掉落模板 (DROP_TEMPLATES)
  └─ 加入 spawnQueue_

Spawner.Update(dt)
  ├─ 同屏存活上限检查 (≥ MAX_ALIVE_ENEMIES 时暂停)
  ├─ 按 spawnInterval 间隔逐个生成
  └─ SpawnFromQueue() → 在屏幕边缘随机位置实例化
```

### 20.2 同屏存活上限

```lua
Config.MAX_ALIVE_ENEMIES = 5  -- 默认同屏最多 5 只存活怪物
```

特殊模式可通过 `maxAliveOverride` 覆盖此上限。

### 20.3 波次配置 (Config.WAVE)

| 字段 | 值 | 说明 |
|------|-----|------|
| `enemiesPerWave` | 5 | 每波基础怪物数 |
| `spawnInterval` | 1.5s | 每只怪物生成间隔 |
| `bossEvery` | 10 | 每 10 波出 Boss |
| `restTime` | 1.5s | 波次间休息时间 |
| `scalingPerWave` | 0.08 | 每波属性增长 8% |

## 21. 战斗流程集成

### 21.1 主循环调用顺序 (BattleSystem.Update)

```
BattleSystem.Update(dt)
  │
  ├─ Spawner.Update()                  ── 生成怪物
  ├─ PlayerAI.Update()                 ── 玩家 AI 攻击
  │
  ├─ EnemySystem.UpdateEnemyDebuffs()  ── Debuff 衰减
  ├─ EnemySystem.UpdateEnemyAI()       ── 移动 + 攻击
  ├─ EnemySystem.UpdateEnemyAbilities()── 技能 + 被动能力
  │
  ├─ BossSkillTemplates.Update()       ── Boss 技能冷却/触发
  ├─ BossSkillTemplates.UpdateProjectiles() ── 弹体碰撞检测
  ├─ BossSkillTemplates.UpdateZones()  ── 区域持续伤害
  ├─ BossSkillTemplates.UpdateDestroyables()── 可摧毁物更新
  │
  ├─ FamilyMechanics.Update()          ── 家族持续效果
  ├─ EliteSystem.UpdateAffixes()       ── 精英词缀持续效果
  │
  └─ StageManager.CheckWaveComplete()  ── 波次完成检测
```

### 21.2 击杀回调链

```
怪物 HP ≤ 0
  ↓
BattleSystem.OnEnemyKilled(enemy)
  ├─ BossSkillTemplates.OnBossDied()     ── Boss 清理 (弹体/区域/可摧毁物)
  ├─ EliteSystem.OnEliteDeath()          ── 精英词缀死亡效果 (爆裂等)
  ├─ FamilyMechanics.OnEnemyDeath()      ── 家族机制 (分裂/复活/碎裂/献祭/潮池/孢子)
  ├─ EnemySystem.OnEnemyDeath()          ── 旧系统 (splitOnDeath/sporeCloud/deathExplode)
  └─ StageManager.OnEnemyKilled()        ── 掉落生成 + 计数 + 波次推进
        └─ GameMode.OnEnemyKilled()      ── 特殊模式适配器 (噩梦/深渊/试炼等)
```

### 21.3 元素类型

```lua
Config.ELEMENTS = {

    baseResist = {
```
