# 装备系统

## 1. 系统概览

装备系统是角色成长的核心循环之一，覆盖 **掉落 → 穿戴 → 强化 → 锻造 → 分解** 的完整生命周期。

**代码结构**（`scripts/state/`）：

| 模块 | 文件 | 职责 |
|------|------|------|
| 编排层 | `Equipment.lua` | 初始化共享上下文，按序安装子模块 |
| 物品工厂 | `equip/ItemFactory.lua` | 装备生成（品质、词缀） |
| 材料管理 | `equip/Materials.lua` | 强化石、魂晶、背包容量 |
| 背包操作 | `equip/Inventory.lua` | 穿戴、排序、锁定、分解 |
| 装备改造 | `equip/Upgrade.lua` | 升级、IP注入、附魔 |
| 锻造系统 | `equip/Forge.lua` | 每日锻造（免费+付费） |
| 运行时迁移 | `equip/Migration.lua` | 旧格式装备自动转换 |
| 词缀查询 | `AffixHelper.lua` | 运行时词缀值汇总 |
| 宝石系统 | `GemSystem.lua` | 镶嵌、拆卸、合成、打孔 |

**关联模块**：

| 模块 | 文件 | 关系 |
|------|------|------|
| 掉落系统 | `battle/Loot.lua` | 战斗中调用 `GenerateEquip` 生成装备掉落 |
| 属性计算 | `state/StatCalc.lua` | 通过 `equipSum()` 汇总装备词缀到角色属性 |
| 存档系统 | `SlotSaveSystem.lua` | 序列化/反序列化装备数据 |

## 2. 品质体系

共 5 个品质等级，品质决定词缀数量、升级上限、IP系数：

| 品质 | 颜色 | 词缀数 | 升级上限 | IP系数 | 掉落权重 |
|------|------|--------|---------|--------|---------|
| 白色 | 灰 | 1 | 0（不可升级） | ×0.50 | 50 |
| 绿色 | 绿 | 2 | +10 | ×0.65 | 30 |
| 蓝色 | 蓝 | 3 | +20 | ×0.80 | 15 |
| 紫色 | 紫 | 4 | +35 | ×0.90 | 4 |
| 橙色 | 橙 | 5 | +50 | ×1.00 | 1 |

**掉落权重调整**：
- 幸运值降低白/绿权重（`×max(0.3, 1-luck)`），提升蓝+品质权重（`×(1+luck×(品质-2))`）
- `lucky_star` 词缀额外加成蓝+品质
- 非Boss击杀有额外压低系数：白×1, 绿×1, 蓝×0.054, 紫×0.02, 橙×0.008

## 3. IP (Item Power) 系统

IP 是装备强度的统一度量，取代旧的 Tier 系统。

### 3.1 IP 计算

```
IP = floor(BaseIP(章节) × 品质系数)
```

- `BaseIP(章节)` — 由 `Config.CalcBaseIP` 按章节计算
- 品质系数 — 见上方品质表
- **升级不改变 IP**（v4.0 起，升级加成独立于 IP 体系）

### 3.2 IP 区间与 Roll 范围

IP 决定词缀数值的 roll 范围（`Config.IP_BRACKETS`）：

| IP 范围 | Roll 下限 | Roll 上限 | 阶段 |
|---------|----------|----------|------|
| ≤150 | 0.15 | 0.30 | 新手期 |
| ≤300 | 0.20 | 0.38 | 开始关注词缀 |
| ≤450 | 0.25 | 0.45 | 学会比较装备 |
| ≤600 | 0.28 | 0.50 | 追求特定词缀 |
| ≤800 | 0.30 | 0.55 | 主线中后期 |
| >800 | 0.33 | 0.60 | 主线天花板 |

词缀最终值公式（`CalcAffixValue`）：

```
value = base × ipFactor × roll
ipFactor = 1 + (IP/100 - 1) × ipScale
```

- `base` — 词缀基础值（`Config.AFFIX_POOL` 中定义）
- `ipScale` — IP 敏感度（0 = 不随 IP 缩放，1.0 = 线性缩放）
- `roll` — 在 [minRoll, maxRoll] 区间内随机

> **ipScale 分层设计**:
> - **ipScale=1.0**（4 条）: `atk`、`hp`、`def`、`hpRegen` — 固定值词缀，随 IP 线性成长
> - **ipScale=0.3~0.7**（8 条）: `critDmg`(0.7)、`fireDmg/iceDmg/lightningDmg/range`(0.5)、`vulnerableDmg`(0.4)、`skillDmg`(0.3)、`elite_hunter`(0.3) — 伤害/范围词缀，适度成长

>
> 详细推导过程见 `docs/newDocs/系统/词缀ipScale平衡推导.md`。

- 仅橙色品质装备可出
- 概率 15%
- 效果：词缀值 ×1.5

### 4.3 词缀与属性计算的关系

`StatCalc` 通过 `equipSum(statKey)` 遍历所有已装备物品，依次汇总：
1. **主属性**（`mainStatId == stat` 时累加 `mainStatValue`）
2. **词缀**（`affixes[]` 按 `id` 匹配求和，值已内含 IP 缩放）
3. **宝石**（`GetGemStats()` 按 stat 匹配求和）

## 6. 主属性系统

每件装备拥有一条**固有主属性**，由槽位决定，不占词缀格。主属性提供稳定的基础数值成长，与词缀的随机 roll 形成互补。

### 6.1 设计理念

- **纵向（Power）**：主属性 + 固定值词缀随 IP 线性成长，保底战力
- **横向（Build Quality）**：百分比词缀 roll 品质决定构建上限
- 主属性是确定性的，没有随机 roll，纯粹跟随 IP 缩放

### 6.2 槽位→主属性映射

| 槽位 | 主属性 | slotBase | 备注 |
|------|--------|----------|------|
| weapon（武器） | ATK | 50 | — |
| gloves（手套） | ATK | 20 | — |
| boots（靴子） | DEF | 30 | — |
| amulet（护符） | HP | 800 | — |
| ring（戒指） | ATK 或 HP | 15 / 400 | 生成时随机二选一 |
| necklace（项链） | hpRegen | 16 | — |

### 6.3 数值公式

```
mainStatValue = slotBase × (IP / 100) + slotBase × 升级等级 × 0.04
```

- 第一项为 IP 线性缩放：IP=100 时主属性值 = slotBase 原值
- 第二项为升级加成：每级增加 slotBase × 4% 的固定值（独立于 IP）
- 满级 50 时额外增加 slotBase × 200% = 2 倍 slotBase
- 注入改变 IP 时，第一项自动重算；升级改变等级时，第二项自动重算

### 6.4 与词缀的区别

| 维度 | 主属性 | 词缀 |
|------|--------|------|
| 数量 | 每件装备固定 1 条 | 1-5 条（按品质） |
| 随机性 | 无随机（槽位决定） | 类型随机 + 数值 roll |
| 占格 | 不占词缀格 | 占词缀格 |
| IP 缩放 | 线性 `slotBase×(IP/100)` | 按 `ipScale` 和 roll 范围 |
| 附魔可洗 | 不可洗 | 可洗 |

### 6.5 存档与序列化

- 持久化字段：`mainStatId`（属性ID）、`mainStatBase`（slotBase）
- `mainStatValue` **不存盘**，每次加载时从 IP + 升级等级 实时重算（`CalcMainStatValueFull`）
- 压缩格式：`msi`（mainStatId）、`msb`（mainStatBase）

### 6.6 旧存档兼容

`Migration.lua` 中的 `ensureMainStat()` 在启动时扫描所有装备：
- 缺少 `mainStatId` 的旧装备自动补充主属性
- 根据 `Config.GetMainStatDef(slot)` 获取定义
- 戒指槽位随机分配 ATK 或 HP

### 6.7 UI 显示

装备卡片中，主属性以金色高亮显示在词缀列表上方：

```
◆ 攻击力 75 [主属性]        ← 金色 (255, 220, 80)
  暴击率 +12.5%              ← 普通词缀
  技能伤害 +8.3%
```

### 6.8 代码入口

| 职责 | 文件 | 函数/字段 |
|------|------|----------|
| 配置 | `Config.lua` | `MAIN_STAT`, `GetMainStatDef()`, `CalcMainStatValue()`, `CalcMainStatValueFull()` |
| 生成 | `equip/ItemFactory.lua` | `GenerateEquip()`, `CreateEquip()` 写入 `mainStatId/Base/Value` |
| 属性汇总 | `state/StatCalc.lua` | `equipSum()` 检查 `item.mainStatId == stat` |
| 升级/注入同步 | `equip/Upgrade.lua` | 升级/注入后用 `CalcMainStatValueFull` 重算 `mainStatValue` |
| UI 显示 | `ui/InventoryCompare.lua` | `BuildHalfCard()`, `BuildDetailCard()` |
| 存档压缩 | `SlotSaveSystem.lua` | `CompressItem` 写 `msi/msb`，`DecompressItem` 恢复并重算 |
| 旧档迁移 | `equip/Migration.lua` | `ensureMainStat()` |

## 9. 装备升级

### 9.1 基本规则

- 白色不可升级，升级上限按品质递增：绿色10级、蓝色20级、紫色35级、橙色50级
- 消耗强化石，费用由 `Config.UpgradeCost(当前等级, 1)` 计算
- 费用公式：`max(2, floor(2 + lv×0.8 + lv²×0.03))`

### 9.2 升级效果

**主属性**（每级提升）：
- 主属性增加 `slotBase × 4%` 的固定值
- 公式：`mainStatValue = slotBase × (IP/100) + slotBase × 升级等级 × 0.04`
- 橙色满级 50 时累计额外增加 2 倍 slotBase

**词缀里程碑**（每5级触发）：
- 每 5 级升级，**随机选一条词缀**获得 +2% 乘算加成
- 每条词缀独立记录被选中次数（`milestoneCount`）
- 公式：`value = baseValue × (1 + milestoneCount × 0.02)`
- 橙色满级 50 时共 10 次里程碑机会，随机分配给各词缀

**升级不改变 IP**：IP 保持不变，升级加成独立于 IP 体系。

### 9.3 分解返还

已升级装备分解时，返还已消耗强化石的 **80%**。

## 10. IP 注入（原 TierUpgrade）

将低章节装备的 IP 提升到当前最高通关章节水平。

### 10.1 前置条件

- 蓝色品质及以上
- 当前 IP < 目标章节对应 IP
- 消耗魔法石（背包道具）

### 10.2 执行逻辑

1. 计算新 IP = `calcItemPower(目标章节, 品质)`（不含升级等级，IP 纯净）
2. 按 `ipScale` 比例缩放所有词缀 baseValue：`newBaseValue = oldBaseValue × (newFactor / oldFactor)`
3. 重新应用各词缀的里程碑倍率：`value = newBaseValue × (1 + milestoneCount × 0.02)`
4. 用 `CalcMainStatValueFull` 重算主属性
5. 消耗魔法石

### 10.3 魔法石类型

- **顶级魔法石** — 目标章节 = 当前最高通关章节
- **普通魔法石** — 目标章节 = 魔法石配置的固定 `targetTier`

## 11. 附魔（洗词缀）

重新 roll 指定位置的词缀。

### 11.1 前置条件

- 紫色品质及以上
- 消耗魂晶

### 11.2 费用公式

```
费用 = floor((50 + IP × 0.5) × 品质倍率)
```

| 品质 | 倍率 |
|------|------|
| 紫色 | ×1 |
| 橙色 | ×2 |

### 11.3 执行逻辑

1. 从槽位词缀池中排除装备上其他已有词缀
2. 从候选池中随机选一条新词缀
3. 按当前 IP 重新 roll 数值（得到 `baseValue`）
4. 保留原词缀的 Greater 状态
5. 新词缀 `milestoneCount` 归零（里程碑积累不继承）
6. 存储 `baseValue` 供后续升级里程碑重算

## 14. 背包管理

### 14.1 容量

- 初始：20 格
- 上限：100 格
- 每次扩容 +4 格，费用递增（100 + 50×(n-1) 魂晶）

- 拾取时品质 ≤ 阈值的装备自动分解为材料

| 品质 | 强化石 |
|------|--------|

| 绿色 | 1 |

`AutoEquipBest` 遍历所有槽位，如果背包中有比当前穿戴 IP 更高的同槽位装备，自动替换。

## 15. 掉落系统

掉落物在战场中的物理表现（`battle/Loot.lua`）：

### 15.1 散落物理

- 掉落物生成时以随机方向散射（速度 100-200）
- 摩擦力衰减，速度 <5 时停止散落
- 散落阶段不触发磁吸

### 15.2 磁吸拾取

- 散落结束后，进入玩家拾取半径即触发磁吸
- 掉落 2秒后 自动磁吸全屏物品
- 磁吸速度随距离递增

### 15.3 背包满处理

- 装备类掉落物背包满时不消失，延长生命值
- 符合自动分解条件的装备仍可被磁吸
- 地面装备堆积上限 100 件，超出时优先移除低品质
