# 材料系统

## 1. 系统概览

材料系统是 D4 风格的多材料体系，取代旧版单一"强化石"。不同品质的装备分解产出不同材料，不同升级等级段消耗不同材料，部分稀有材料需要通过特殊副本获取。

**设计理念**：
- **分层进阶**：低级材料从普通分解获得，高级材料从高品质分解或副本掉落获取
- **消耗递进**：升级低等级消耗基础材料，高等级消耗稀有材料，最高等级额外需要副本材料
- **多来源多用途**：每种材料同时服务于升级、锻造等多个系统

**代码结构**：

| 模块 | 文件 | 职责 |
|------|------|------|
| 材料定义 | `Config.lua` | 6 种材料属性、分解产出表、升级消耗分段、锻造消耗 |
| 消耗公式 | `ConfigCalc.lua` | `UpgradeCost()`、`GetForgeMaterialCost()` |
| 数据 API | `state/equip/Materials.lua` | 通用增删查扣 API + 旧版兼容 |
| 数据初始化 | `GameState.lua` | 材料字段初始化、离线奖励分解逻辑 |
| 存档迁移 | `SlotSaveSystem.lua` | v12→v13 旧档 stone→iron 迁移 |

**关联模块**：

| 模块 | 文件 | 关系 |
|------|------|------|

## 2. 材料定义

### 2.1 六种材料

| 材料 ID | 名称 | 颜色 | 稀有度 | 主要来源 | 主要用途 |
|---------|------|------|--------|---------|---------|
| `iron` | 锈蚀铁块 | `{180, 160, 140}` | common | 分解白/绿装 | Lv0-9 升级、锻造 |
| `crystal` | 暗纹晶体 | `{100, 140, 220}` | uncommon | 分解蓝装 | Lv10-19 升级、锻造 |
| `wraith` | 怨魂碎片 | `{180, 80, 220}` | rare | 分解紫装、深渊模式 | Lv20-34 升级、锻造 |
| `eternal` | 永夜之魂 | `{255, 165, 0}` | legendary | 分解橙装、试炼/秘境 | Lv35+ 升级 |
| `abyssHeart` | 深渊之心 | `{200, 50, 80}` | mythic | 深渊模式 Boss | Lv45+ 额外消耗 |
| `riftEcho` | 裂隙残响 | `{80, 220, 200}` | mythic | 试炼/秘境/世界Boss | 预留（高端消耗） |

**定义位置**: `Config.lua` → `Config.MATERIAL_DEFS`

**快查表**: `Config.MATERIAL_MAP[matId]` → 返回材料定义 `{ id, name, color, rarity, desc }`

**ID 列表**: `Config.MATERIAL_IDS = { "iron", "crystal", "wraith", "eternal", "abyssHeart", "riftEcho" }`

### 2.2 材料层级设计

```
                 ┌─────────────┐
                 │  深渊之心    │ ← 深渊模式 Boss 独占
                 │  裂隙残响    │ ← 试炼/秘境/世界Boss
                 └──────┬──────┘
                        │ mythic (副本掉落)
         ┌──────────────┴──────────────┐
         │          永夜之魂            │ ← 分解橙装 + 试炼/秘境
         └──────────┬──────────────────┘
                    │ legendary
         ┌──────────┴──────────────────┐
         │          怨魂碎片            │ ← 分解紫装 + 深渊层通关
         └──────────┬──────────────────┘
                    │ rare
         ┌──────────┴──────────────────┐
         │          暗纹晶体            │ ← 分解蓝装
         └──────────┬──────────────────┘
                    │ uncommon
         ┌──────────┴──────────────────┐
         │          锈蚀铁块            │ ← 分解白/绿装
         └─────────────────────────────┘
                    common
```

## 3. 分解产出

分解装备时根据品质获得对应材料。

**定义位置**: `Config.lua` → `Config.DECOMPOSE_MATERIALS`

| 品质 | qualityIdx | 产出材料 |
|------|-----------|---------|
| 白色 | 1 | 锈蚀铁块 ×1 |
| 绿色 | 2 | 锈蚀铁块 ×3 |
| 蓝色 | 3 | 锈蚀铁块 ×2 + 暗纹晶体 ×2 |
| 紫色 | 4 | 暗纹晶体 ×2 + 怨魂碎片 ×3 |
| 橙色 | 5 | 怨魂碎片 ×2 + 永夜之魂 ×2 |

**已升级装备额外返还**: 分解已升级装备时，返还已投入材料的 **80%**（`Config.UPGRADE_REFUND_RATIO = 0.8`），通过 `item.upgradeMatSpent` 记录的各材料消耗量计算。

**分解代码入口**: `state/equip/Inventory.lua` → `DecomposeItem(invIndex)`

返回值: `gold, matsTable`

```lua
local gold, mats = GameState.DecomposeItem(invIndex)
-- gold: number (金币)
-- mats: { [matId] = amount } (材料表)
```

## 4. 升级消耗

装备升级根据当前等级段消耗不同材料。

### 4.1 材料分段

**定义位置**: `Config.lua` → `Config.UPGRADE_MATERIAL_TIERS`

| 升级等级范围 | 主消耗材料 | 说明 |
|------------|-----------|------|
| Lv 0 ~ 9 | 锈蚀铁块 (iron) | 低级升级阶段 |
| Lv 10 ~ 19 | 暗纹晶体 (crystal) | 中级升级阶段 |
| Lv 20 ~ 34 | 怨魂碎片 (wraith) | 高级升级阶段 |
| Lv 35+ | 永夜之魂 (eternal) | 顶级升级阶段 |

### 4.2 深渊之心额外消耗

**Lv 45+** 升级时，每次额外消耗 **1 个深渊之心** (`Config.UPGRADE_ABYSS_HEART_LEVEL = 45`)。

### 4.3 消耗公式

**位置**: `ConfigCalc.lua` → `Config.UpgradeCost(level, chapter)`

```lua
-- 基础数量
base = max(2, floor(2 + level × 0.8 + level² × 0.03))
amount = floor(base × chapter)

-- 确定主材料
matId = 根据 UPGRADE_MATERIAL_TIERS 按等级段查找

-- 返回材料表
result = { [matId] = amount }

-- 高等级额外消耗
if level >= 45 then
    result.abyssHeart = 1
end
```

**返回值**: `{ [matId] = amount, [abyssHeart] = 1? }` — 材料消耗表

### 4.4 消耗数值示例

以 chapter=1 为例：

| 等级 | 材料 | 数量 | 深渊之心 |
|------|------|------|---------|
| 0 | iron | 2 | - |
| 5 | iron | 6 | - |
| 10 | crystal | 11 | - |
| 20 | wraith | 22 | - |
| 35 | eternal | 39 | - |
| 45 | eternal | 57 | +1 |
| 50 | eternal | 67 | +1 |

## 5. 锻造消耗

锻造装备消耗多种材料。

**定义位置**: `Config.lua` → `Config.FORGE_MATERIAL_COST` / `Config.FORGE_MATERIAL_COST_LOCK`

| 资源 | 普通锻造 | 锁定部位 |
|------|---------|---------|
| 锈蚀铁块 (iron) | 30 | 60 |
| 暗纹晶体 (crystal) | 15 | 30 |
| 怨魂碎片 (wraith) | 5 | 10 |
| 金币 | 30 × √scaleMul | 60 × √scaleMul |

**API**: `Config.GetForgeMaterialCost(lockSlot)` — 返回 `{ iron=N, crystal=N, wraith=N }` 材料消耗表

## 6. 副本掉落

### 6.1 深渊模式 (AbyssMode)

**文件**: `AbyssMode.lua` → `OnEnemyKilled()`

| 条件 | 掉落材料 | 数量公式 |
|------|---------|---------|
| Boss 击杀 | 深渊之心 | `1 + floor(floor / 10)` |
| Boss 击杀 | 怨魂碎片 | `2 + floor(floor / 5)` |
| 普通层通关 | 怨魂碎片 | `1 + floor(floor / 10)` |

*注: `floor` 指当前深渊层数*

### 6.2 试炼之塔 (SpireTrial)

**文件**: `SpireTrial.lua` → `M.Complete()`

| 条件 | 掉落材料 | 数量公式 |
|------|---------|---------|
| 试炼通关 | 裂隙残响 | `spireId × 2` |
| 试炼通关 | 永夜之魂 | `spireId` |

*spireId: 1=初级试炼, 2=中级试炼, 3=高级试炼*

### 6.3 套装秘境 (SetDungeon)

**文件**: `SetDungeon.lua` → `EndFight(won)`

| 条件 | 掉落材料 | 数量 |
|------|---------|------|
| 普通模式胜利 | 永夜之魂 | 1 |
| 困难模式胜利 | 永夜之魂 | 2 |
| 困难模式胜利 | 裂隙残响 | 1 |

### 6.4 世界 Boss (WorldBoss)

**文件**: `WorldBoss.lua`

**参战奖励** (`GrantParticipationReward()`):
- 每次参战: 裂隙残响 ×1

**赛季排名奖励** (`RANK_REWARDS`):

| 排名 | 裂隙残响 | 永夜之魂 |
|------|---------|---------|
| 第 1 名 | 3 | 2 |
| 第 2 名 | 2 | 1 |
| 第 3 名 | 1 | 1 |
| 第 4-50 名 | 1 | - |

### 6.5 每日奖励 (DailyRewards)

**文件**: `DailyRewards.lua`

每日签到奖励中包含材料，具体数量按签到天数配置在 `DAILY_REWARDS` 表中。

### 6.6 掉落来源汇总

| 材料 | 分解来源 | 副本来源 | 其他来源 |
|------|---------|---------|---------|
| 锈蚀铁块 | 白/绿/蓝装分解 | - | 广告兑换、兑换码、每日奖励 |
| 暗纹晶体 | 蓝/紫装分解 | - | 广告兑换、兑换码、每日奖励 |
| 怨魂碎片 | 紫/橙装分解 | 深渊模式 | 每日奖励 |
| 永夜之魂 | 橙装分解 | 试炼/秘境 | 每日奖励 |
| 深渊之心 | - | 深渊模式 Boss | - |
| 裂隙残响 | - | 试炼/秘境/世界Boss | - |

## 7. 数据 API

**文件**: `state/equip/Materials.lua`

所有 API 通过 `M.Install(GS, ctx)` 注入到 `GameState` 表上。

### 7.1 通用材料 API

| API | 签名 | 说明 |
|-----|------|------|
| `AddMaterial` | `(matId, amount)` | 增加指定材料 |
| `GetMaterial` | `(matId) → number` | 获取指定材料数量 |
| `AddMaterials` | `(matTable)` | 批量增加材料 `{ [id]=amount }` |
| `HasMaterials` | `(matTable) → ok, reason` | 检查是否有足够材料 |
| `SpendMaterials` | `(matTable) → ok, reason` | 批量扣除材料（先检查再扣） |

### 7.2 旧版兼容 API

| API | 实际映射 |
|-----|---------|
| `AddStone(amount)` | → `AddMaterial("iron", amount)` |
| `GetStone()` | → `GetMaterial("iron")` |

### 7.3 使用示例

```lua
-- 检查并消耗升级材料
local matCost = Config.UpgradeCost(currentLevel, chapter)
local ok, reason = GameState.HasMaterials(matCost)
if ok then
    GameState.SpendMaterials(matCost)
end

-- 分解装备获得材料
local gold, mats = GameState.DecomposeItem(invIndex)
-- mats = { iron = 2, crystal = 2 }  -- 蓝装分解示例

-- 批量增加副本掉落
GameState.AddMaterials({ abyssHeart = 1, wraith = 3 })
```

## 8. 存储与序列化

### 8.1 运行时数据

材料存储在 `GameState.materials` 表中，所有 7 个字段受 SecureNum 保护：

```lua
GameState.materials = {
    iron       = 0,  -- 锈蚀铁块
    crystal    = 0,  -- 暗纹晶体
    wraith     = 0,  -- 怨魂碎片
    eternal    = 0,  -- 永夜之魂
    abyssHeart = 0,  -- 深渊之心
    riftEcho   = 0,  -- 裂隙残响
    soulCrystal = 0, -- 魂晶 (独立于材料体系)
}
```

### 8.2 存档版本

当前存档版本: **v13** (`CURRENT_SAVE_VERSION = 13`)

存档序列化: `SlotSaveSystem.lua` → `SerializeState()` / `DeserializeState()`

### 8.3 装备材料消耗记录

已升级装备通过 `upgradeMatSpent` 字段记录各材料累计消耗：

```lua
item.upgradeMatSpent = { iron = 30, crystal = 45 }  -- 消耗记录
```

压缩存档字段: `ums`（upgradeMatSpent）

## 9. 存档迁移 (v12 → v13)

**迁移文件**: `SlotSaveSystem.lua` → `MIGRATIONS[12]`

迁移执行两步操作：

### 9.1 材料迁移

```
旧版: materials.stone = N
新版: materials.iron = N (1:1 映射)
```

同时初始化 6 个新材料字段为 0。

### 9.2 装备消耗记录迁移

```
旧版: item.upgradeStonesSpent = N
新版: item.upgradeMatSpent = { iron = N }
```

遍历所有已装备和背包中的装备，将旧版单一消耗记录转换为多材料消耗表。

### 9.3 兼容性保障

- 旧版 `AddStone()` / `GetStone()` API 自动映射到 `iron`
- `Config.DECOMPOSE_STONES` 保留用于历史迁移参考
- 旧版 `Config.FORGE_STONE_COST` 等字段保留供可能的旧 UI 引用
