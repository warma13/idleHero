# 噩梦地牢 - 设计文档

> 终局装备刷取副本，灵感源自暗黑4噩梦地下城。通过消耗钥石进入随机词缀增强的高难度地牢，层数越高、词缀越多、掉落越好。

---

## 1. 设计目标

| 目标 | 说明 |
|------|------|
| **终局装备出口** | 主线天花板 roll 范围 0.33-0.60，噩梦地牢将 roll 上限推至 0.60-1.00 |
| **可重复挑战** | 不限每日次数（钥石驱动），鼓励长时间刷取 |
| **渐进难度** | 1-100 层递进，玩家自由选择挑战层级 |
| **战术深度** | 随机词缀（增益+惩罚）改变每次通关策略 |
| **材料循环** | 高层产出稀有材料（深渊之心、裂隙残响），驱动终局强化和锻造 |

---

## 2. 解锁条件

| 条件 | 值 | 说明 |
|------|------|------|
| 最低章节 | 通关第 7 章 | 终局内容，玩家已理解装备词缀系统 |
| 世界层级 | 无限制 | 地牢内部有独立难度缩放 |

## 3. 钥石系统

### 3.1 钥石获取

钥石（Sigil）是进入噩梦地牢的消耗品，属于 bagItem 类型。

| 来源 | 掉落规则 |
|------|---------|
| 主线 Boss（10/20/30...关） | 通关第 7 章后，每个 Boss 25% 概率掉落 Lv1 钥石 |
| 深渊模式 | 每 10 层必掉 1 枚，层级 = floor(深渊层/10) |
| 无尽试炼 | 每 10 层 50% 概率掉落，层级 = floor(试炼层/15) |
| 世界 Boss | 参与奖固定 1 枚，层级 = floor(maxChapter/2) |
| 噩梦地牢通关 | 100% 掉落下一级钥石（层级+1~3 随机） |
| 套装秘境困难 | 通关 30% 概率掉落 Lv1 钥石 |

### 3.2 钥石属性

```lua
-- bagItem 结构
{
    itemId = "nightmare_sigil",
    count  = 1,
    -- 附加数据存在 extra 中
    extra  = {
        tier     = 5,                      -- 地牢层级 (1-100)
        affixes  = { "empowered", "frostborn" },  -- 预置词缀ID (1-4 条)
        element  = "fire",                 -- 主题元素 (决定怪物外观+背景)
    },
}
```

### 3.3 钥石层级与词缀数

| 层级范围 | 词缀数 | 说明 |
|---------|--------|------|
| 1-10 | 1 正面 + 0 负面 | 入门期，仅有增益 |
| 11-25 | 1 正面 + 1 负面 | 开始有惩罚 |
| 26-50 | 2 正面 + 1 负面 | 中期 |
| 51-75 | 2 正面 + 2 负面 | 高难度 |
| 76-100 | 2 正面 + 3 负面 | 极限挑战 |

## 4. 词缀系统

### 4.1 正面词缀（玩家增益）

| ID | 名称 | 效果 |
|----|------|------|
| `empowered` | 力量涌动 | 玩家攻击力 +20% |
| `swiftness` | 疾风之力 | 玩家攻速 +25% |
| `fortified` | 铁壁 | 玩家防御力 +30% |
| `vampiric` | 吸血本能 | 玩家生命偷取 +3% |
| `critical_mass` | 暴击聚能 | 玩家暴击率 +15%，暴击伤害 +20% |
| `elemental_fury` | 元素狂怒 | 玩家全元素增伤 +15% |

### 4.2 负面词缀（玩家惩罚 / 怪物增强）

| ID | 名称 | 效果 |
|----|------|------|
| `armored` | 铁甲 | 怪物防御力 +50% |
| `frenzied` | 狂暴 | 怪物攻速 +40%，攻击力 +15% |
| `shadowborn` | 暗影降生 | 怪物数量 +30% |
| `frostborn` | 寒霜 | 玩家移速 -20%，攻速 -10% |
| `corrosive` | 腐蚀 | 玩家防御力 -25%，生命回复 -30% |
| `afflicted` | 诅咒 | 玩家受到的元素伤害 +25% |
| `suppression` | 压制 | 玩家暴击率上限降至 30% |
| `desolate` | 荒芜 | 生命偷取效率 -50% |

### 4.3 词缀生成规则

1. 钥石生成时随机抽取词缀，正面和负面分别从对应池中不重复抽取
2. 通关奖励的下一层钥石继承 0-1 条当前词缀，其余重新随机
3. 同一钥石不会出现互相矛盾的词缀（如 `empowered` + `suppression`）

## 5. 地牢结构

### 5.1 三阶段制

每个噩梦地牢由 3 个阶段组成，必须按顺序清除：

| 阶段 | 名称 | 内容 | 限时 |
|------|------|------|------|
| 1 | 扫荡 | 清除 N 波小怪+精英 | 无（整体限时统一） |
| 2 | 精英层 | 强化精英群，含冠军怪 | — |
| 3 | 守护者 | 1 个 Boss + 增援 | — |

**整体限时**：`180 + tier * 2` 秒（Lv1 = 182 秒，Lv50 = 280 秒，Lv100 = 380 秒）

### 5.2 怪物缩放

```
-- 怪物基础 scaleMul 锚定于玩家最高章节的 Boss 缩放
baseScale = StageConfig.GetBossScaleMul(maxChapter, 10)

-- 层级加成 (渐近线模型，避免后期数值爆炸)
tierMul = 1.0 + 2.0 * tier / (tier + 50)
-- Lv1  → 1.04
-- Lv10 → 1.33
-- Lv25 → 1.67
-- Lv50 → 2.00
-- Lv75 → 2.20
-- Lv100 → 2.33

-- 词缀加成
affixMul = 1.0 + 负面词缀带来的怪物增益

-- 最终怪物 scaleMul
scaleMul = baseScale * tierMul * affixMul
```

### 5.3 怪物数量

| 阶段 | 普通怪 | 精英 | 冠军 | Boss |
|------|--------|------|------|------|
| 1 | 15 + tier/5 | 2 + tier/15 | 0 | 0 |
| 2 | 8 + tier/10 | 3 + tier/10 | 1 + tier/25 | 0 |
| 3 | 4 + tier/20 | 0 | 0 | 1 |

（所有值向下取整）

## 6. 掉落与奖励

### 6.1 装备掉落

噩梦地牢是终局装备的核心出口，roll 范围突破主线天花板：

| 层级范围 | minRoll | maxRoll | 保底品质 | 装备数量 |
|---------|---------|---------|---------|---------|
| 1-10 | 0.40 | 0.65 | 蓝色 | 2-3 件 |
| 11-25 | 0.45 | 0.72 | 蓝色 | 3-4 件 |
| 26-50 | 0.50 | 0.80 | 紫色 | 3-5 件 |
| 51-75 | 0.55 | 0.88 | 紫色 | 4-5 件 |
| 76-100 | 0.60 | 1.00 | 紫色 | 4-6 件 |

**装备生成规则**：
- IP (Item Power) = 玩家当前平均 IP × (1 + tier × 0.005)，上限受 maxChapter 约束
- Boss 阶段必掉 1 件橙色装备（Lv50+ 必掉 2 件）
- 品质权重受层级影响：层级越高，橙色概率越高

### 6.2 品质权重（Boss 掉落）

| 层级范围 | 蓝色 | 紫色 | 橙色 |
|---------|------|------|------|
| 1-10 | 50% | 40% | 10% |
| 11-25 | 30% | 50% | 20% |
| 26-50 | 15% | 50% | 35% |
| 51-75 | 5% | 45% | 50% |
| 76-100 | 0% | 35% | 65% |

### 6.3 材料掉落

| 材料 | 条件 | 数量 |
|------|------|------|
| 锈蚀铁块 | 所有层级 | 10 + tier/2 |
| 暗纹晶体 | Lv10+ | 5 + tier/4 |
| 怨魂碎片 | Lv25+ | 2 + tier/8 |
| 永夜之魂 | Lv50+ | 1 + tier/20 |
| 深渊之心 | Lv75+ Boss 必掉 | 1（Lv90+ 为 2） |
| 裂隙残响 | Lv50+ Boss 30% 概率 | 1 |

### 6.4 钥石递进

通关后 100% 获得下一级钥石：
- 层级 = 当前层级 + random(1, 3)
- 上限 100

## 9. 技术实现方案

### 9.1 文件结构

| 文件 | 职责 |
|------|------|
| `scripts/NightmareDungeon.lua` | 核心逻辑：状态管理、战斗入口/退出、怪物生成、词缀应用、阶段推进、HUD 绘制、GameMode 适配器 |
| `scripts/NightmareDungeonConfig.lua` | 词缀定义、层级缩放公式、掉落表、怪物配置 |
| `scripts/ui/NightmareDungeonPanel.lua` | 入口面板：钥石选择、词缀预览、难度信息、进入按钮 |
| `scripts/ui/NightmareDungeonResult.lua` | 结算界面：通关时间、掉落展示、钥石递进预览 |
| `scripts/Config.lua` → `Config.NIGHTMARE_DUNGEON` | 全局配置常量 |

### 9.2 GameMode 适配器接口

```lua
-- 注册
GameMode.Register("nightmareDungeon", adapter)

-- 适配器实现 (关键方法)
adapter.OnEnter         -- 消耗钥石、初始化阶段、应用词缀
adapter.OnExit          -- 恢复关卡状态
adapter.BuildSpawnQueue -- 根据阶段+层级构建怪物队列
adapter.GetBattleConfig -- 返回 Boss 限时、计时器配置
adapter.OnEnemyKilled   -- 阶段击杀计数、阶段推进判断
adapter.CheckWaveComplete -- 全灭检测 → 进入下一阶段 or 通关
adapter.OnDeath         -- 死亡 → 失败结算（不扣钥石，但不给奖励）
adapter.OnTimeout       -- 超时 → 失败结算
adapter.IsTimerMode     -- return true (限时模式)
adapter.GetDisplayName  -- "噩梦地牢 Lv.{tier}"
adapter.DrawWaveInfo    -- 绘制阶段进度 + 层级 + 词缀图标
```

### 9.3 词缀运行时应用

```lua
-- 进入地牢时：
function ApplyAffixes(affixes)
    for _, affixId in ipairs(affixes) do
        local def = AFFIX_DEFS[affixId]
        if def.target == "player" then
            -- 直接修改 playerBattle 的 buff 系统
            PlayerBuff.Add(affixId, def.stats, FIGHT_DURATION)
        elseif def.target == "monster" then
            -- 存储到运行时状态，BuildSpawnQueue 时应用
            runtime.monsterMods[affixId] = def.stats
        end
    end
end
```

### 9.4 钥石物品配置

```lua

    name     = "噩梦钥石",

    color    = { 180, 60, 220 },

```

### 9.5 与现有系统的集成点

| 系统 | 集成方式 |
|------|---------|
| `BattleSystem` | 通过 GameMode 适配器，无需修改战斗核心 |
| `StageManager` | `OnEnemyKilled` 中被 GameMode 拦截 |
| `Loot` | 通关结算时调用 `Loot.Spawn()` 批量生成装备掉落 |
| `DropManager` | Boss 击杀时委托 DropManager 处理，注入高 roll 范围 |
| `GameState.CreateEquip` | 装备生成时传入 nightmareDungeon 专属 roll 范围参数 |
| `ChallengePanel / 主界面` | 添加入口按钮 |
| `SaveSystem` | 注册 `nightmareDungeon` 存档域 |

## 13. 数值平衡要点

### 13.1 装备强度递进

```
主线极限 (Ch12 Boss):    roll 0.33~0.60, IP ≈ 800
噩梦地牢 Lv1:            roll 0.40~0.65, IP ≈ 810    -- 略高于主线
噩梦地牢 Lv50:           roll 0.50~0.80, IP ≈ 1000   -- 明显优势
噩梦地牢 Lv100:          roll 0.60~1.00, IP ≈ 1200   -- 理论极限
```

### 13.2 难度曲线

- Lv1-10: 比主线 Boss 略强，新手友好
- Lv25: 约等于主线 1.67 倍难度，需要蓝/紫装毕业
- Lv50: 约等于主线 2 倍难度，需要橙装 + 套装效果
- Lv75+: 终极挑战，需要精通词缀组合 + 完善 build
- Lv100: 理论上限，仅供顶尖玩家挑战

### 13.3 钥石经济

- 平均每次主线 Boss 击杀获得 0.25 枚钥石
- 深渊 10 层 = 1 枚钥石，相当于约 10 分钟产出
- 通关噩梦地牢后必得下一级钥石，形成自循环
- 失败不消耗钥石（鼓励尝试高层）

### P0 - 核心功能
1. NightmareDungeonConfig.lua - 词缀定义、缩放公式
2. NightmareDungeon.lua - GameMode 适配器、阶段管理
3. Config.NIGHTMARE_DUNGEON - 配置常量

### P1 - UI 与集成
5. NightmareDungeonPanel.lua - 入口面板、钥石选择
6. NightmareDungeonResult.lua - 结算界面
7. ChallengePanel / 主界面入口按钮
8. 钥石掉落集成（主线 Boss、深渊、试炼等）

### P2 - 打磨
9. 词缀视觉反馈（HUD 图标 + 战斗特效）
10. 排行榜（最高层级、最佳通关时间）
11. 成就系统集成
