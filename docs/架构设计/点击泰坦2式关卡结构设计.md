# 点击泰坦2式关卡结构设计

> **目标**: 将现有"章节×关卡"有限推进系统改造为"无限线性关卡 + Boss门 + 转生循环"结构
>
> **状态**: 设计草案 v1

---

## 一、现有系统 vs TT2 模式对比

| 维度 | 现有系统 | TT2 模式 |
|------|---------|---------|
| **关卡结构** | 16章 × 10关，手工配置怪物 | 单轴无限关卡（第1关→第∞关） |
| **怪物来源** | StageConfig 手写每关每波 | 公式生成 + 模板轮换 |
| **数值膨胀** | scaleMul 对数缩放 | 指数缩放，保证必定卡关 |
| **Boss 机制** | 每5关一个Boss，有计时器 | 每N关一个Boss，**限时30秒** |
| **卡关后** | 原地重试（刷关模式） | 可选择**转生**重来 |
| **永久成长** | 装备/技能/属性点（不重置） | 转生货币 → 永久倍率加成 |
| **进度感** | 章节推进（有终点） | 每次转生推得更远（无终点） |

prestigeStage = {

## 三、关卡生成公式

### 3.1 怪物数量（每关需击杀数）

```
killTarget(stage) =
    stage ≤ 5  → 5                           -- 前5关新手引导，少怪
    stage ≤ 50 → 8 + floor(stage / 10)       -- 8~13只，缓增
    else       → 10                          -- 稳定在10只
```

Boss 关固定 1 只 Boss（击杀即通关）。

### 3.2 怪物血量缩放（核心公式）

```
monsterHP(stage) = baseHP × (1 + stage × 0.1) × 1.07^stage

-- 示例:
-- stage=1:   baseHP × 1.1  × 1.07  ≈ baseHP × 1.18
-- stage=50:  baseHP × 6.0  × 29.5  ≈ baseHP × 177
-- stage=200: baseHP × 21.0 × 7979  ≈ baseHP × 167K
-- stage=500: baseHP × 51.0 × 5.4e15 → 天文数字，必须转生
```

关键：`1.07^stage` 是指数项，**保证玩家伤害增长终究会追不上**。

### 3.3 怪物攻击力缩放

```
monsterATK(stage) = baseATK × (1 + stage × 0.05) × 1.05^stage
```

ATK 增速略慢于 HP，避免玩家被秒杀导致体验差。重点靠"打不过"（时间不够）而非"被打死"来逼迫转生。

### 3.4 Boss 频率与计时器

```
isBoss(stage) = (stage % BOSS_INTERVAL == 0)

BOSS_INTERVAL = 5               -- 每5关一个Boss
BOSS_TIMER    = 30               -- Boss限时30秒（可随转生次数增加）
BOSS_HP_MUL   = 10               -- Boss血量 = 同关普通怪 × 10
BOSS_ATK_MUL  = 2.5              -- Boss攻击 = 同关普通怪 × 2.5
```

### 3.5 怪物模板轮换（利用现有资源）

不需要手写每关怪物，从现有 `StageConfig.MONSTERS` 中按阶段轮换：

```
主题轮换周期 = 50 关
themeIndex(stage) = floor((stage - 1) / 50) % themeCount + 1

STAGE_THEMES = {
    [1] = { monsters = {"ash_rat","void_bat","bandit"},         boss = "boss_ash_lord",     name = "灰烬荒原" },
    [2] = { monsters = {"frost_imp","ice_wraith","snow_wolf"},  boss = "boss_frost_lord",   name = "冰封深渊" },
    [3] = { monsters = {"swamp_frog","spore_shroom","rot_worm"},boss = "boss_swamp_king",   name = "腐朽沼泽" },
    -- ... 对应现有16章主题，循环使用
}
```

每50关切换一次视觉主题和怪物种类，数值由公式生成，主题只决定外观和抗性。

## 四、转生（Prestige）系统

### 4.1 触发条件

- 玩家可在**任意时刻**手动触发转生（但必须推到第 20 关以上才有收益）
- 推荐时机：Boss 卡关打不过时

### 4.2 转生获得圣物数量

```
relicsGained(maxStage) =
    maxStage < 20  → 0                              -- 太早转生无收益
    maxStage < 100 → floor(maxStage / 10)            -- 2~9 个
    else           → floor(maxStage^0.6)             -- 渐缓增长
```

推得越远收益越高，但**边际递减**——避免"永远不转生"的策略碾压。

### 4.3 转生时重置的内容

| 重置 | 保留 |
|------|------|
| 当前关卡 → 第1关 | 转生次数、历史最高关 |
| 英雄等级 → 1级 | 神器及其等级 |
| 金币 → 0 | 圣物（转生货币） |
| 技能点 → 未分配 | 装备（可选：保留/重置） |
| 当前 Buff 清空 | 技能树解锁状态（可选） |

### 4.4 神器升级

神器提供永久乘算加成，用圣物升级：

```
artifactCost(level) = costBase × level^1.5

artifactBonus(level) = base × level
-- 例如: 狂怒之剑 lv10 → atkMul = 0.05 × 10 = +50% 攻击力
```

所有神器加成是**乘算**叠加，不是加算：

```
totalAtkMul = 1.0
for _, art in ipairs(ownedArtifacts) do
    if art.stat == "atkMul" then
        totalAtkMul = totalAtkMul × (1 + art.base × art.level)
    end
end
-- 3个 +50% 的神器 → 1.5 × 1.5 × 1.5 = 3.375 倍（不是 2.5 倍）
```

## 五、玩家伤害成长来源

为了和怪物指数增长赛跑，玩家伤害需要有多个增长维度：

| 来源 | 增长方式 | 转生时 |
|------|---------|--------|
| 英雄等级 | 线性（每级 +ATK） | 重置 |
| 装备 | 随机掉落，tier 缩放 | 可选保留 |
| 技能树 | 技能点加成 | 可选保留 |
| 属性点 | 手动分配 | 重置 |
| **神器** | 永久乘算倍率 | **保留** |
| **转生次数加成** | 每次转生 +2% 基础攻击 | **保留** |

关键设计：转生后前期关卡"秒杀"感来自神器的乘算倍率。玩家 lv1 的基础攻击被神器放大后，前几十关的怪物 HP 不堪一击。

## 六、与现有系统的集成方案

### 方案 A：新增 GameMode（推荐）

在现有 `GameMode` 注册表中新增 `prestige` 模式适配器，**不改动现有章节系统**：

```
涉及文件:
├── PrestigeMode.lua          # GameMode 适配器（新建）
│   ├── BuildSpawnQueue()     # 从公式+模板生成怪物
│   ├── OnEnemyKilled()       # 计数 → 通关判定
│   ├── OnTimeout()           # Boss 超时 → 卡关
│   ├── OnDeath()             # 死亡 → 回到当前关重试
│   └── OnNextWave()          # 推进到下一关
│
├── PrestigeConfig.lua        # 数值公式+神器定义（新建）
│   ├── CalcMonsterHP(stage)
│   ├── CalcRelicsGained(maxStage)
│   ├── ARTIFACTS 定义
│   └── STAGE_THEMES 轮换表
│
├── state/PrestigeState.lua   # 转生状态管理（新建，Install模式注入GameState）
│   ├── DoPrestige()          # 执行转生
│   ├── UpgradeArtifact(id)   # 升级神器
│   ├── GetArtifactBonus(stat)# 查询总加成
│   └── GetRelicsCost(id,lv)  # 升级费用
│
├── ui/PrestigePage.lua       # 转生UI（新建）
│   ├── 转生按钮 + 预览收益
│   ├── 神器列表 + 升级
│   └── 历史统计
│
├── SlotSaveSystem.lua        # 新增域注册 prestige（改动）
├── GameState.lua              # 新增 prestige 数据初始化（改动）
├── BattleSystem.lua           # GameMode 分支已有，无需大改
└── main.lua                   # 入口切换（改动）
```

优势：
- 现有16章推图完全保留，两套模式共存
- 复用 GameMode 适配器架构，battle/ 子系统零改动
- 可以做成"解锁内容"——通关第N章后解锁转生模式

### 方案 B：替换主线推进

将整个主线从"章节制"改为"无限关卡制"。**不推荐**，因为会丢失16章的精心设计内容。

## 七、战斗流程（方案 A 下）

```
进入转生模式
  │
  ▼
┌─────────────────────────────────────────────┐
│ 第 N 关（普通关）                             │
│                                              │
│  Spawner 生成 killTarget 只怪（公式算HP/ATK）  │
│  ↓                                           │
│  玩家自动战斗，逐只击杀                        │
│  ↓                                           │
│  击杀数 >= killTarget → 自动进入第 N+1 关      │
│  死亡 → 原地重试第 N 关                        │
└──────────────────┬──────────────────────────┘
                   │
          stage % 5 == 0?
          ┌────────┴────────┐
          │ 否               │ 是
          ▼                 ▼
     下一普通关       ┌─────────────────────────┐
                     │ 第 N 关（Boss关）         │
                     │                          │
                     │  生成 1 只 Boss           │
                     │  启动 30 秒倒计时         │
                     │  ↓                       │
                     │  30秒内击杀 → 通关，继续   │
                     │  超时 → 卡关，停在此关     │
                     │  死亡 → 原地重试           │
                     │                          │
                     │  ★ 卡关时：显示转生按钮    │
                     └──────────┬──────────────┘
                                │
                       玩家点击转生?
                       ┌────────┴────────┐
                       │ 否               │ 是
                       ▼                 ▼
                   继续尝试        ┌───────────────────┐
                   （提升伤害）     │ 转 生              │
                                  │                    │
                                  │ 计算圣物奖励        │
                                  │ 重置关卡/等级/金币   │
                                  │ 保留神器+圣物        │
                                  │ → 回到第 1 关       │
                                  └───────────────────┘
```

## 十、数值平衡要点

### 10.1 两条曲线赛跑

```
玩家DPS增长: 多项式 × 神器乘算
  ≈ level^1.5 × artifactMul × equipMul × skillMul

怪物HP增长: 指数
  ≈ baseHP × 1.07^stage

关键: 指数必然超越多项式。
每次转生，神器乘算提升 → 多项式的"系数"变大 → 能多推几十关。
但 1.07^stage 的指数底不变 → 终归会再次卡关。
这就是循环的数学保证。
```

### 10.2 转生收益曲线（参考数据）

| 转生次数 | 累计圣物 | 典型神器等级 | 约能推到 | 体感 |
|---------|---------|------------|---------|------|
| 0 | 0 | 0 | ~30关 | 初见卡关 |
| 1 | ~3 | 1-2 | ~50关 | 明显变强 |
| 3 | ~15 | 3-5 | ~100关 | 前50关秒杀 |
| 5 | ~40 | 5-8 | ~180关 | 开始感受指数墙 |
| 10 | ~120 | 10-15 | ~300关 | 中期稳定循环 |
| 20 | ~400 | 20-25 | ~500关 | 深度养成 |
| 50 | ~2000 | 40+ | ~1000关 | 长期目标 |

### 10.3 防刷机制

- 转生最低门槛 20 关（防止无意义快速转生刷圣物）
- 圣物公式用 `stage^0.6`（亚线性），推得越远边际递减
- 神器升级费用用 `level^1.5`（超线性），后期越来越贵
- 总效果：最优策略是"推到卡关附近 → 转生"，不是"刷第1关"或"死磕不转生"

## 十一、实现优先级

### P0 - 核心循环（先能玩起来）

1. `PrestigeConfig.lua` — 数值公式和常量
2. `state/PrestigeState.lua` — 转生状态 + GameState 注入
3. `PrestigeMode.lua` — GameMode 适配器（怪物生成、击杀计数、Boss计时、关卡推进）
4. HUD 改动 — 显示关卡号、击杀进度、Boss 倒计时
5. SlotSaveSystem 域注册 — 存档

**验收标准**: 能进入转生模式，自动战斗推关，Boss 限时，转生后重置回第1关，神器提升伤害。

### P1 - 体验打磨

6. 转生面板 UI — 预览收益、神器升级
7. 转生动画 — 屏幕特效、数字飞入动画
8. 主题轮换 — 每50关切换背景和怪物外观
9. 离线收益适配 — 离线时自动推关（按 DPS 和怪物 HP 公式计算能推到哪关）

### P2 - 深度扩展

10. 更多神器（20-50个）
11. 氏族（公会）系统 — 转生时额外加成
12. 锦标赛 — 固定时间窗口内比谁推得远
13. 阶梯奖励 — 首次到达某关卡的一次性奖励（如第100/200/500关）

## 十二、风险与注意事项

| 风险 | 说明 | 应对 |
|------|------|------|
| 数值爆炸 | float64 在 ~stage 700+ 可能溢出 | 预留 BigNum 或科学计数法显示（现有 Utils.FormatNumber 已支持 `×10^N`） |
| 与章节模式冲突 | 两套模式的装备/等级/属性点是否共享 | 建议独立存档域，互不影响 |
| 转生动力不足 | 玩家不理解"为什么要重来" | 首次卡关时强引导，展示转生后的预期收益 |
| 前期太无聊 | 转生后前几十关都是秒杀 | 加速机制：已通关的关卡可一键跳过或10倍速 |
| Boss 计时太紧/太松 | 30秒可能不适合所有阶段 | 可以随 stage 微调：`timer = 30 + floor(stage / 100)` |
