# P1 属性系统重构设计

> **目标**: 将现有 10 个独立加点属性折叠为 D4 式 4 核心属性 (STR/DEX/INT/WIL)
> **前置**: P0 怪物家族已完成
> **后续**: P2 词缀系统、P3 技能系统

---

## 1. 痛点分析

### 1.1 现状

当前系统有 **10 个独立可加点属性**:

| # | key | label | 效果 | 缩放 |
|---|-----|-------|------|------|
| 1 | atk | 攻击力 | +6×attrScale 每点 | attrScale |
| 2 | spd | 攻速 | +0.05/s 每点 (DR) | 无 |
| 3 | crit | 暴击率 | +2% 每点 | 无 |
| 4 | critDmg | 暴击伤害 | +10% 每点 | 无 |
| 5 | range | 攻击范围 | +8 每点 (DR) | 无 |
| 6 | luck | 幸运 | +3% 每点 (掉落品质) | 1.5^ch |
| 7 | hp | 生命 | +30×attrScale 每点 + 0.3%MaxHP | attrScale |
| 8 | vit | 体力 | def+0.5×attrScale, hpRegen+1.5, healPct+0.8%, shldPct+0.5% | attrScale(def) |
| 9 | ten | 韧性 | +2%减益抗性, cap 80% | 无 |
| 10 | mastery | 精通 | CDR+1.5%(DR), skillDmg+1% | 无 |

**问题**:
1. **选择瘫痪**: 10 个维度，玩家不知道该点哪个，导致均匀分配 → 失去build差异化
2. **数值碎片化**: 每个属性效果单一(除 vit/mastery 外)，缺乏"一个选择带来多种收益"的爽感
3. **无职业差异基础**: 当前术士是唯一职业，10 属性对所有未来职业都一样，无法体现职业特色
4. **每级 3 点分 10 项**: 每次升级的加点感受稀薄

### 1.2 D4 方案优势

D4 的 4 核心属性 (STR/DEX/INT/WIL):
- **每个属性同时给多种收益** → 选择有意义
- **通用效果 + 职业专属效果** → 同一属性不同职业体验不同
- **只有 4 个选项** → 决策聚焦，build 差异明显

## 2. 四属性定义

### 2.1 属性总览

| 属性 | ID | 中文 | 通用效果 (每点) | 术士专属效果 (每点) |
|------|----|----|----------------|-------------------|
| **力量** | STR | 力量 | +1 护甲 (Armor) | +0.1% 生命上限 |
| **敏捷** | DEX | 敏捷 | +0.025% 闪避 | +0.02% 暴击率 |
| **智力** | INT | 智力 | +0.05% 全抗 | +0.1% 技能伤害 |
| **意志** | WIL | 意志 | +0.1% 治疗效果, +0.25% 超杀伤害 | +0.1% 资源恢复 (CDR) |

> **主属性**: 术士的主属性是 **INT** (智力)，给予技能伤害加成。
> **多职业扩展**: 后续新增职业时，只需定义该职业的 `classBonus` 表即可。

### 2.2 通用效果详解

#### 力量 (STR) — 护甲

```
Armor = STR × 1.0 × attrScale
DEF = baseDEF + levelDEF + Armor + equipDEF
```

- 直接叠加到总防御力 (`GetTotalDEF`)
- 替代原 vit 的 `def+0.5×attrScale` 部分
- 术士专属: +0.1% 生命上限 (叠入 hpPct 乘区)

#### 敏捷 (DEX) — 闪避

```
dodgeChance = DEX × 0.025%  (cap 30%)
```

- **新机制**: 受到伤害时有概率完全闪避 (在 DamagePlayer 中插入)
- 术士专属: +0.02% 暴击率 (叠入 `GetCritRate`)

#### 智力 (INT) — 全抗

```
allResist = INT × 0.05%  (cap 40%)
```

- **全抗**: 所有元素抗性统一加成，在伤害公式的抗性乘区生效
- 术士专属 (主属性): +0.1% 技能伤害 (叠入 `GetSkillDmg`)

#### 意志 (WIL) — 治疗 + 超杀

```
healBonus = WIL × 0.1%       → GetHealMul 乘区
overkillDmg = WIL × 0.25%    → 新增 GetOverkillDmg
```

- **治疗效果**: 替代原 vit 的 `healPct+0.8%` 部分
- **超杀伤害**: 对生命值低于 30% 的敌人额外伤害 (新机制, 在 DamageFormula 中插入)
- 术士专属: +0.1% 资源恢复 → 映射为 CDR 加成 (叠入 `GetSkillCdMul`)

### 2.3 职业扩展设计

```lua
-- Config.lua 新增
Config.CLASS_BONUS = {
    warlock = {
        STR = { stat = "hpPct",    perPoint = 0.001 },  -- +0.1% HP
        DEX = { stat = "crit",     perPoint = 0.0002 }, -- +0.02% 暴击率
        INT = { stat = "skillDmg", perPoint = 0.001 },  -- +0.1% 技能伤害
        WIL = { stat = "cdr",      perPoint = 0.001 },  -- +0.1% CDR
    },
    -- 未来: warrior, rogue, druid ...
}
Config.CURRENT_CLASS = "warlock"
```

新增职业时只需添加一项 `CLASS_BONUS[className]`，无需修改属性计算逻辑。

## 3. 旧 → 新属性映射

### 3.1 十属性归并表

| 旧属性 | 归并到 | 映射方式 |
|--------|--------|---------|
| **atk** | *(移除加点)* | 攻击力不再来自加点，改为纯装备+技能+称号 |
| **spd** | *(移除加点)* | 攻速不再来自加点，改为纯装备+套装+技能 |
| **crit** | DEX | DEX 职业效果提供暴击率 |
| **critDmg** | *(移除加点)* | 暴伤不再来自加点，改为纯装备+溢出暴击 |
| **range** | *(移除加点)* | 范围不再来自加点，改为纯装备 |
| **luck** | *(移除加点)* | 幸运不再来自加点，改为纯装备 |
| **hp** | STR | STR 职业效果提供生命加成 |
| **vit** | STR + WIL | def→STR, healPct→WIL, hpRegen→装备, shldPct→装备 |
| **ten** | *(移除加点)* | 韧性不再来自加点，改为纯装备/称号 |
| **mastery** | INT + WIL | skillDmg→INT, CDR→WIL |

### 3.2 设计哲学

**核心原则**: 加点 = 4 大属性 (战略选择)，细分属性 = 装备/技能/称号 (战术构建)

移除 6 项加点属性 (atk/spd/critDmg/range/luck/ten) 不会让它们消失，只是不再从"加点"获得。玩家仍然通过装备词缀、套装效果、技能树、称号来获取这些属性。这使得：
1. 加点决策更有深度（4 选项，每个影响多种属性）
2. 装备/build 构建更重要（细分属性差异化来自装备选择）

## 4. 数值设计

### 4.1 每级点数

```lua
Config.POINTS_PER_LEVEL = 3  -- 保持不变
```

每升一级获得 3 点，分配到 4 个属性中。与 D4 一致(D4 每级 2 点分 4 项)，但本游戏由于挂机特性，保留 3 点使加点感更强。

### 4.2 通用效果数值

```lua
Config.CORE_ATTR = {
    STR = {
        armor_per_point = 1.0,       -- 每点+1 Armor (×attrScale)
        label = "力量",
        desc = "护甲 +%d | %s",      -- 通用 | 职业
    },
    DEX = {
        dodge_per_point = 0.00025,   -- 每点+0.025% 闪避
        dodge_cap = 0.30,            -- 闪避上限 30%
        label = "敏捷",
        desc = "闪避 +%.2f%% | %s",
    },
    INT = {
        all_resist_per_point = 0.0005, -- 每点+0.05% 全抗
        all_resist_cap = 0.40,         -- 全抗上限 40%
        label = "智力",
        desc = "全抗 +%.2f%% | %s",
    },
    WIL = {
        heal_per_point = 0.001,      -- 每点+0.1% 治疗效果
        overkill_per_point = 0.0025, -- 每点+0.25% 超杀伤害
        overkill_hp_threshold = 0.30, -- 敌人HP < 30% 触发超杀
        label = "意志",
        desc = "治疗 +%.1f%% 超杀 +%.1f%% | %s",
    },
}
```

### 4.3 边际递减保留

| 机制 | 变化 |
|------|------|
| 攻速 DR | **保留**，不受属性重构影响 (来源=装备+套装+技能) |
| 范围 DR | **保留**，同上 |
| CDR DR | **保留**，WIL 的 CDR 效果也走 DR 曲线 |
| 闪避 cap | **新增**, 硬顶 30% |
| 全抗 cap | **新增**, 硬顶 40% |

### 4.4 attrScale 适用规则

```
STR 护甲: Armor = pts × 1.0 × attrScale    ← 跟随章节缩放
DEX 闪避: dodgeChance = pts × 0.025%        ← 不缩放 (百分比)
INT 全抗: allResist = pts × 0.05%           ← 不缩放 (百分比)
WIL 治疗: healBonus = pts × 0.1%            ← 不缩放 (百分比)
WIL 超杀: overkillDmg = pts × 0.25%         ← 不缩放 (百分比)
```

只有 **STR 的护甲** 需要 attrScale 缩放（因为 DEF 是绝对值，需要跟随章节增长）。其他三项都是百分比效果，天然平衡。

## 5. 新机制详设

### 5.1 闪避 (Dodge)

**触发点**: `Combat.lua → DamagePlayer()`

```lua
-- 在 DEF 计算之前插入
local dodgeChance = GameState.GetDodgeChance()
if dodgeChance > 0 and math.random() < dodgeChance then
    -- 闪避成功: 不受任何伤害
    -- 显示 "闪避" 飘字
    return 0
end
```

**韧性交互**: 闪避不受韧性影响（闪避的是直接伤害，不是 debuff）。

**装备扩展**: 未来可在装备词缀中加入 `dodge` 属性，叠加到 DEX 的闪避。

### 5.2 全元素抗性 (All Resist)

**触发点**: `DamageFormula.lua → 抗性乘区`

```lua
-- 现有: 单元素抗性 (火抗/冰抗/毒抗/奥术抗/水抗)
-- 新增: 全抗加成
local allResist = GameState.GetAllResist()
local specificResist = equipSum(elemResKey) -- 如 "fireRes"
local totalResist = specificResist + allResist
-- 应用到 Config.ResistMul(totalResist)
```

**与现有单抗的关系**: 全抗与单抗**加算**，共享 ResistMul 曲线。

### 5.3 超杀伤害 (Overkill)

**触发点**: `DamageFormula.lua → 新增乘区 (Crit 之后, DEF 之前)`

```lua
-- 当目标生命低于阈值时触发
local overkillMul = 1.0
if target.hp / target.maxHP < Config.CORE_ATTR.WIL.overkill_hp_threshold then
    overkillMul = 1.0 + GameState.GetOverkillDmg()
end
-- finalDmg = baseDmg × critMul × overkillMul × defMul × resistMul
```

**设计意图**: 鼓励 WIL 型 build 在收割阶段更强，与 STR(硬抗) / DEX(闪避) / INT(技能伤害) 形成差异化。

## 7. 存档迁移

### 7.1 迁移策略: v6 → v7

旧存档:
```json
{
  "allocatedPoints": {
    "atk": 20, "spd": 15, "crit": 10, "critDmg": 8,
    "range": 5, "luck": 5, "hp": 12, "vit": 10, "ten": 5, "mastery": 10
  },
  "freePoints": 0
}
```

新存档:
```json
{
  "allocatedPoints": {
    "STR": 0, "DEX": 0, "INT": 0, "WIL": 0
  },
  "freePoints": 100
}
```

### 7.2 迁移规则

**全额退还**: 将旧 10 属性的所有已分配点数全部归还为 `freePoints`，让玩家重新分配。

```lua
-- SlotSaveSystem.lua 迁移函数
local function migrateV6ToV7(data)
    local old = data.allocatedPoints
    if not old then return end
    -- 计算总点数
    local totalPts = 0
    for _, v in pairs(old) do totalPts = totalPts + v end
    -- 全部归还
    data.freePoints = (data.freePoints or 0) + totalPts
    -- 重建为新 4 属性
    data.allocatedPoints = { STR = 0, DEX = 0, INT = 0, WIL = 0 }
end
```

**理由**: 旧属性与新属性没有精确对应关系，自动映射会导致玩家 build 与预期不符。全额退还 + 弹窗提示是最安全的方案。

### 7.3 首次登录提示

迁移完成后设置标记 `_attrMigrated = true`，首次进入游戏时弹窗:

```
属性系统已升级！

你的 {totalPts} 点属性点已全部退还。
新系统使用 4 大核心属性: 力量/敏捷/智力/意志
请前往角色页重新分配属性点。

[前往分配]
```

### 8.1 CharacterPage 属性 Tab 改版

[icon] 攻击力   1234  [-10][-][  20 ][+][+10]

├─────────────────────────────────────────────┤

### 8.2 UI 改版要点

1. **主行**: 属性名 + 已分配点数 + 加减按钮
2. **子行**: 灰色小字显示该属性的所有效果 (通用+职业)，实时计算
3. **一键加点**: 下拉菜单选择分配策略 (全INT / 均分 / 自定义比例)
4. **批量操作**: 保留 +10/-10 按钮
5. **重置**: 保留现有魂晶消耗机制

### 8.3 战力公式调整

```lua
-- 旧: 10 属性加权求和
-- 新: 4 属性的派生效果加权
GameState.GetPower = function()
    local atk = GameState.GetTotalAtk()      -- 装备+技能
    local spd = GameState.GetAtkSpeed()       -- 装备+技能
    local crit = GameState.GetCritRate()      -- DEX + 装备
    local critDmg = GameState.GetCritDmg()    -- 装备 + 溢出
    local range = GameState.GetRange()        -- 装备
    local luck = GameState.GetLuck()          -- 装备
    local maxHP = GameState.GetMaxHP()        -- STR + 装备
    local def = GameState.GetTotalDEF()       -- STR + 装备
    local dodge = GameState.GetDodgeChance()  -- DEX
    local allRes = GameState.GetAllResist()   -- INT
    local skillDmg = GameState.GetSkillDmg()  -- INT + 装备
    local overkill = GameState.GetOverkillDmg() -- WIL
    local elemDmg = GameState.GetElemDmg()
    local reactionDmg = GameState.GetReactionDmgFromEquip()
    return math.floor(
        atk * 2 + spd * 50 + crit * 100 + critDmg * 40
        + range * 1.5 + luck * 80
        + maxHP * 0.1 + def * 3
        + dodge * 500 + allRes * 300
        + skillDmg * 80 + overkill * 60
        + elemDmg * 60 + reactionDmg * 60
    )
end
```

## 10. 风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
