# 折光矿脉 — 怪物设计文档

> 版本: v1.0
> 日期: 2026-03-19
> 对应代码: `scripts/ResourceDungeon.lua` → `BuildMineQueue()`

---

## 一、设计目标

折光矿脉是**每日限时资源副本**（60秒，每日3次），核心目标是快节奏刷取宝石和棱镜。怪物设计应服务于以下体验：

1. **快节奏击杀**: 怪物不应过于耐打，玩家应能在60秒内击杀大量怪物
2. **适度威胁**: 怪物需要有一定威胁（尤其精英），但不应频繁击杀玩家
3. **矿脉主题**: 怪物外观和命名围绕"矿脉/晶体/矿石"主题，与章节推进贴图复用
4. **与章节难度对齐**: scaleMul = 章节末关 × 0.9，略低于玩家当前实力上限

## 二、当前实现分析

### 常量

| 参数 | 值 | 说明 |
|------|------|------|
| NORMAL_COUNT | 30 | 普通怪数量 |
| ELITE_SPAWN_AT | 15 | 第15只后插入精英 |
| ELITE_HP_MUL | 3.0 | 精英HP倍率 |
| MONSTER_SCALE | 0.9 | 相对章节末关的强度系数 |
| FIGHT_DURATION | 60 | 战斗时长(秒) |

### 当前怪物配置（问题）

```lua
BEHAVIOR_POOL = { "bruiser", "swarm", "glass", "swarm", "bruiser" }
-- 精英: "tank"
-- 抗性: 全部 "balanced"
-- 名称: 全部 "矿脉守卫" / "矿脉精英"
```

**问题**:
1. 行为模板仅用3种（bruiser/swarm/glass），缺乏变化
2. 抗性全部balanced，无元素策略深度
3. 名称统一为"矿脉守卫"，缺乏辨识度
4. 精英仅1只tank，缺少精英多样性
5. 无能力标签，战斗体验单调

## 三、改进方案

### 3.1 怪物阵容设计

矿脉怪物分为**三个梯队**，按出场顺序排列：

| 梯队 | 出场位置 | 数量 | 行为模板 | 设计意图 |
|------|---------|------|---------|---------|
| 先锋 | #1~#10 | 10 | swarm × 6, glass × 4 | 快速清扫，建立节奏 |
| 主力 | #11~#20 | 10 | bruiser × 4, debuffer × 3, caster × 2, exploder × 1 | 增加难度，需要策略 |
| 精锐 | #21~#31 | 10+1精英 | swarm × 3, bruiser × 3, glass × 2, tank × 1, exploder × 1 + **精英** | 最后冲刺，高压收尾 |

**精英怪**: 第16只位置（主力梯队中段），行为模板 = **caster**（远程输出型），HP × 3.0

> 精英改为caster而非tank的理由：
> - tank精英太肉，60秒限时中浪费大量时间
> - caster精英有威胁（高ATK + isRanged + healAura）但HP不至于磨不动
> - 如果保留tank精英，应降低ELITE_HP_MUL至2.0

### 3.2 行为模板分配序列

```lua
-- 新 BEHAVIOR_POOL (31只怪，含1精英)
MINE_QUEUE = {
    -- 先锋 (1-10): 快节奏
    "swarm", "swarm", "glass", "swarm", "glass",
    "swarm", "glass", "swarm", "glass", "swarm",
    -- 主力 (11-15): 增加难度
    "bruiser", "debuffer", "caster", "bruiser", "debuffer",
    -- 精英 (16): caster精英
    "ELITE_caster",
    -- 主力续 (17-20): 
    "exploder", "bruiser", "debuffer", "caster",
    -- 精锐 (21-31): 混合高压
    "swarm", "bruiser", "glass", "tank", "swarm",
    "bruiser", "exploder", "glass", "swarm", "bruiser", "glass",
}
```

### 3.3 抗性模板分配

矿脉怪物的抗性**跟随玩家当前最高通关章节的主题元素**，而非固定balanced：

| 行为模板 | 抗性模板规则 | 说明 |
|---------|-------------|------|
| swarm | 章节主题元素抗性 | 常规跟随 |
| glass | all_low | 脆皮快杀 |
| bruiser | balanced | 中庸，任何元素都能打 |
| debuffer | 章节主题元素抗性 | 同swarm |
| caster | 章节主题元素抗性 | 需要克制元素 |
| tank | phys_armor | 物理硬但魔法弱 |
| exploder | all_low | 鼓励快速击杀 |
| **精英** | 章节主题元素抗性 | 迫使玩家注意元素克制 |

**章节主题元素 → 抗性模板映射**:

| 章节主题元素 | 对应抗性模板 |
|------------|-------------|
| fire | fire_res |
| ice | ice_res |
| poison | poison_res |
| water | water_res |
| arcane | arcane_res |
| holy | holy_res |

```lua
-- 伪代码
local theme = MonsterTemplates.ChapterThemes[chapter]
local themeResistId = theme.element .. "_res"  -- "fire_res", "ice_res" 等
-- holy → holy_res
```

### 3.4 能力标签配置

矿脉怪物的能力标签**等级统一为1级**（最低），避免矿脉怪物比章节正式怪物更强。

| 行为模板 | 携带标签 | 说明 |
|---------|---------|------|
| swarm | packBonus Lv1 (C≥2) | 群狼加成，数量多时有威胁 |
| glass | defPierce Lv1 (C≥2) | 穿甲，快速造成伤害 |
| bruiser | slowOnHit Lv1 | 命中减速 |
| debuffer | slowOnHit Lv1 | 核心控制 |
| caster | isRanged, healAura Lv1 (C≥4) | 远程+少量群体回复 |
| tank | antiHeal, hpRegen Lv1 (C≥2) | 肉盾+微量回血 |
| exploder | deathExplode Lv1 | 死亡爆炸 |
| **精英(caster)** | isRanged, healAura Lv2, lifesteal Lv1 | 高威胁远程精英 |

> 标签等级固定为1（精英healAura=2），不随章节递增。矿脉的难度增长完全由scaleMul驱动。

### 3.5 矿脉主题命名

怪物名称围绕**矿脉/晶体/矿石**主题，按行为模板分配：

| 行为模板 | 矿脉名称 | 设计意图 |
|---------|---------|---------|
| swarm | 碎晶虫 | 晶体矿脉中的虫群 |
| glass | 折光蝠 | 利用光线折射的快速刺客 |
| bruiser | 矿脉卫兵 | 矿脉中的正规守卫 |
| debuffer | 辉石蛞蝓 | 散发减速黏液的矿脉生物 |
| caster | 晶能术士 | 操控晶体能量的远程法师 |
| tank | 岩晶巨兽 | 全身覆盖矿晶的重型生物 |
| exploder | 爆晶虫 | 体内蓄满不稳定晶体能量 |
| **精英** | **折光领主** | 矿脉主宰 |

## 四、数值预期

### 4.1 强度校准

矿脉怪物 scaleMul = `StageConfig.CalcScaleMul(maxChapter, 10) × 0.9`

以 swarm 基准值 (hp=35, atk=3) 为例:

| 玩家章节 | scaleMul × 0.9 | swarm HP | swarm ATK (^0.75) | 预期击杀速度 |
|---------|---------------|----------|-------------------|-------------|
| Ch1 | ~7.0 | 245 | ~13 | 很快 |
| Ch4 | ~10.4 | 364 | ~17 | 快 |
| Ch8 | ~13.5 | 472 | ~20 | 中等 |
| Ch12 | ~13.2 | 462 | ~20 | 中等 |

> 玩家实际DPS远超这些HP值（已通关该章节末关的前提），30只普通怪+1精英应能在60秒内全部清除。

### 4.2 精英数值

精英(caster): hp=70 × scaleMul × 3.0, atk=15 × scaleMul^0.75

| 玩家章节 | 精英HP | 精英ATK | 说明 |
|---------|--------|---------|------|
| Ch1 | ~1470 | ~66 | 需要几秒集火 |
| Ch8 | ~2835 | ~92 | 需要明显集火 |
| Ch12 | ~2772 | ~90 | 同上 |

> caster精英自带isRanged，会在远处输出。healAura Lv2 每7秒为周围怪物回复5% HP，构成一定威胁。

### 4.3 60秒击杀数预期

| 玩家实力 | 预期击杀数 | 说明 |
|---------|----------|------|
| 刚通关该章 | 20~25 / 31 | 可能来不及清完 |
| 超过该章1-2章 | 28~31 / 31 | 基本全清 |
| 超过该章3+章 | 31 / 31 | 轻松全清 |

> 这与奖励公式 CalcRewards(C, K, eliteKilled) 配合：击杀越多，宝石越多。

## 五、贴图复用策略

矿脉怪物**复用章节贴图**（已有 `CHAPTER_MOB_SPRITES` 表），按 `maxChapter % 12` 循环选取：

```lua
local chapter = ((maxChapter - 1) % 12) + 1
local sprites = CHAPTER_MOB_SPRITES[chapter]
```

7种行为模板 → 4张贴图轮换分配。未来如有专属矿脉贴图（晶体/矿石风格），可新增 `MINE_SPRITES` 表替换。

## 六、实现方案（代码变更）

### 需修改文件

仅 `ResourceDungeon.lua` 的 `BuildMineQueue()` 函数。

### 变更点

1. **替换 BEHAVIOR_POOL**: 用新的 `MINE_QUEUE` 序列（见 3.2 节）
2. **抗性模板**: 根据行为模板 + 章节主题选择（见 3.3 节），替换固定 "balanced"
3. **能力标签**: 通过 `opts.tags` 传入（见 3.4 节）
4. **命名**: 使用矿脉主题名称（见 3.5 节），替换固定 "矿脉守卫"
5. **精英行为**: 从 "tank" 改为 "caster"，或保留 "tank" 但降低 ELITE_HP_MUL 至 2.0

### 伪代码

```lua
function ResourceDungeon.BuildMineQueue()
    local C = GameState.records.maxChapter or 1
    local scaleMul = StageConfig.CalcScaleMul(C, 10) * MONSTER_SCALE

    local chapter = ((C - 1) % 12) + 1
    local sprites = getChapterSprites(chapter)
    local theme = MonsterTemplates.ChapterThemes[chapter]
    local themeResistId = theme.element .. "_res"

    -- 怪物定义表: { behaviorId, resistId, name, tags }
    local MINE_DEFS = {
        { "swarm",    themeResistId, "碎晶虫",   { packBonus = (C >= 2) and 1 or nil } },
        { "glass",    "all_low",     "折光蝠",   { defPierce = (C >= 2) and 1 or nil } },
        { "bruiser",  "balanced",    "矿脉卫兵", { slowOnHit = 1 } },
        { "debuffer", themeResistId, "辉石蛞蝓", {} },  -- slowOnHit 已是默认标签
        { "caster",   themeResistId, "晶能术士", { healAura = (C >= 4) and 1 or nil } },
        { "tank",     "phys_armor",  "岩晶巨兽", { hpRegen = (C >= 2) and 1 or nil } },
        { "exploder", "all_low",     "爆晶虫",   {} },  -- deathExplode 已是默认标签
    }

    -- 出场序列 (索引对应 MINE_DEFS)
    local SEQUENCE = {
        1,1,2,1,2, 1,2,1,2,1,       -- 先锋 (swarm+glass)
        3,4,5,3,4,                   -- 主力前半
        0,                           -- 精英 (特殊处理)
        7,3,4,5,                     -- 主力后半
        1,3,2,6,1, 3,7,2,1,3,2,     -- 精锐
    }

    local queue = {}
    for i, defIdx in ipairs(SEQUENCE) do
        if defIdx == 0 then
            -- 精英
            local template = MonsterTemplates.Assemble("caster", themeResistId, chapter, {
                name = "折光领主",
                image = sprites[1],
                tags = { healAura = 2, lifesteal = 1 },
            })
            template.hp = template.hp * ELITE_HP_MUL
            template.isElite = true
            template.color = { 255, 180, 50 }
            table.insert(queue, { templateId = "elite_mine", template = template, scaleMul = scaleMul, expScaleMul = 0 })
        else
            local def = MINE_DEFS[defIdx]
            local spriteIdx = ((i - 1) % #sprites) + 1
            local template = MonsterTemplates.Assemble(def[1], def[2], chapter, {
                name = def[3],
                image = sprites[spriteIdx],
                tags = def[4],
            })
            table.insert(queue, { templateId = def[1] .. "_mine_" .. i, template = template, scaleMul = scaleMul, expScaleMul = 0 })
        end
    end

    return queue
end
```
