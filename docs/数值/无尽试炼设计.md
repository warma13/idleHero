# 无尽试炼 — 怪物设计与经验掉落

> 版本: v1  
> 日期: 2026-03-15  
> 对应代码: `scripts/EndlessTrial.lua`, `scripts/MonsterTemplates.lua`, `scripts/battle/Spawner.lua`  
> 关联文档: `怪物模板系统.md`, `经验与等级公式.md`, `数值锚定与战力曲线.md`

---

## §1 设计目标

无尽试炼是独立于章节推图的**核心练级内容**，定位为"高难度、高收益"模式。

| 维度 | 章节推图 | 无尽试炼 |
|------|---------|---------|
| 目标 | 推进剧情，解锁新内容 | 提升等级，检验 build 质量 |
| 难度曲线 | 固定（设计好的关卡序列） | 递增（每层 +6%，无上限） |
| 怪物来源 | `StageConfig.MONSTERS` 硬编码 | `MonsterTemplates.Assemble()` 动态组装 |
| 经验收益 | 正常（`expScaleMul = scaleMul`） | 高（取消 `^0.3` 惩罚，额外倍率加成） |
| 经验发放 | 逐怪掉落 | 通关当层时一次性发放，每层只能领取一次 |
| 结束条件 | 通关（固定波次） | 永久推进（每层一次性通关，死亡可重试） |
| 排行榜 | 无 | 最高层数（衡量 build 质量的连续指标） |

**核心理念**: 试炼是永久推进的楼层系统。每层通关给一次丰厚经验，死亡不给经验但可重试。build 越强，能推进的层数越多，从试炼获得的总经验越多。

## §2 难度缩放（scaleMul）

### 2.1 现状问题

当前试炼 scaleMul 与玩家进度完全脱钩：

```lua
-- 当前公式（EndlessTrial.lua）
scaleMul = 2.0 × 1.25^(floor - 1)
```

- 固定基础值 2.0，不考虑玩家当前章节
- Ch1 玩家和 Ch12 玩家面对同样的 floor 1 难度
- 每层增长 25%（过陡），10 层后 scaleMul ≈ 18.6，绝大多数玩家 15 层内死亡

### 2.2 新设计：锚定玩家章节

```
baseScaleMul = StageConfig.CalcScaleMul(playerChapter, playerStage)
floorScaleMul = baseScaleMul × 1.06^(floor - 1)
```

| 参数 | 值 | 说明 |
|------|---|------|
| `baseScaleMul` | 玩家当前章节的 scaleMul | 通过 `StageConfig.CalcScaleMul()` 获取，包含 `difficultyRatio × tierMul × playerPowerMul` |
| 每层增长率 | 6% | 温和递增，支持深推 |
| Boss 层加成 | ×2.0 | 每 10 层 Boss，额外翻倍 |

### 2.3 层数与倍率关系

以 6% 增长率为例，`floorMul = 1.06^(floor - 1)`：

| 层数 | floorMul | 含义 |
|------|----------|------|
| 1    | 1.00×    | 与当前章节同等难度 |
| 10   | 1.69×    | 比当前章节难 69% |
| 20   | 3.03×    | 约 3 倍难度 |
| 30   | 5.42×    | 约 5.4 倍 |
| 50   | 17.3×    | 约 17 倍 |
| 75   | 72.1×    | 约 72 倍 |
| 100  | 301×     | 约 301 倍 |

**设计意图**：
- 普通玩家（章节 build 刚好及格）预期推到 10-20 层
- 优化过 build 的玩家推到 30-50 层
- 极限 build 玩家推到 50-100+ 层
- 排行榜层数成为 build 质量的**连续度量**，区分度极高

## §3 怪物生成：MonsterTemplates 接入

### 3.1 现状问题

当前试炼从硬编码列表循环选怪：

```lua
-- 当前: 73 个普通怪 ID + 14 个 Boss ID 循环
EndlessTrial.NORMAL_MONSTERS = { "ash_rat", "rot_worm", ... }
```

- 怪物种类固定，高层体验单调
- 无能力标签（tag）叠加，机制复杂度不随层数增长
- 抗性由独立的 `RESIST_ROTATION` 表覆盖，与 MonsterTemplates 的抗性体系割裂

### 3.2 新设计：动态组装

每层怪物通过 `MonsterTemplates.Assemble()` 动态生成：

```lua
local def = MonsterTemplates.Assemble(behaviorId, resistId, chapter, {
    tags = { slowOnHit = 2, packBonus = 3 },
})
```

### 3.3 行为模板选取策略

7 种行为模板按层数**轮换 + 混搭**，确保机制多样性：

| 层数区间 | 每层怪物组成 | 设计意图 |
|---------|-------------|---------|
| 1-10    | 单一行为模板，每层换一种 | 教学期，让玩家逐一认识每种行为 |
| 11-30   | 2 种行为混搭（主 70% + 辅 30%） | 组合期，出现坦克+远程等经典搭配 |
| 31-60   | 2-3 种行为混搭 | 挑战期，需要应对多种机制并存 |
| 60+     | 3 种行为混搭，偶尔全 tank/全 glass 等极端层 | 极限期，考验 build 极限 |

**行为模板轮换序列**（参考）：

```
主行为循环: swarm → bruiser → glass → debuffer → tank → caster → exploder → (循环)
辅行为: 从剩余模板中随机选取，避免与主行为重复
```

Boss 层（每 10 层）：独立 Boss 池，不使用普通行为模板。保留当前 Boss 循环机制，未来可扩展为 MonsterTemplates Boss 模板。

### 3.4 抗性模板轮换策略

保留当前"每 N 层切换一组抗性"的**轮换机制**，但将抗性来源从独立的 `RESIST_ROTATION` 硬编码表替换为 `MonsterTemplates.Resists` 的 12 种抗性模板。

**轮换规则**：
- 轮换周期：每 `RESIST_ROTATION_PERIOD`（默认 5）层切换一组
- 轮换序列：12 种抗性模板按固定顺序循环
- 同一周期内所有层共享相同的 resistId

**轮换序列**（参考）：

```
balanced → fire_res → ice_res → poison_res → water_res → arcane_res →
holy_res → phys_armor → magic_vuln → all_low → all_high → mixed_a → (循环)
```

| 层数 | 轮换组 | resistId | 玩家体验 |
|------|-------|----------|---------|
| 1-5  | 第 1 组 | `balanced` | 无明显弱点/抗性，各系 build 均可 |
| 6-10 | 第 2 组 | `fire_res` | 火系伤害被削，其他系有优势 |
| 11-15 | 第 3 组 | `ice_res` | 冰系伤害被削 |
| ... | ... | ... | 依次轮换 |
| 56-60 | 第 12 组 | `mixed_a` | 火抗+水弱，不对称克制 |
| 61-65 | 第 13 组 | `balanced`（循环回第 1 组） | 新一轮循环 |

**抗性系数（resistCoeff）**：试炼不绑定章节主题，统一使用固定 resistCoeff，按层数区间递增：

| 层数区间 | resistCoeff | 等效章节 | 说明 |
|---------|-------------|---------|------|
| 1-10    | 0.3         | Ch1-2   | 抗性数值低，弱点/抗性感知弱 |
| 11-30   | 0.5         | Ch3-4   | 开始有明显的元素克制感 |
| 31-60   | 0.7-0.85    | Ch5-7   | 非对应元素 build 感受到压力 |
| 60+     | 1.0-1.1     | Ch9-12  | 满系数，抗性成为硬门槛 |

**设计意图**：
- 轮换提供**可预判性**——玩家记住序列后可提前规划
- resistCoeff 递增确保低层轮换只是"风味差异"，高层轮换成为"生死差异"
- 12 种模板轮换一圈 = 60 层，比当前 6 组 × 5 层 = 30 层的周期更长，高层不易重复

### 3.5 能力标签叠加策略

能力标签（tags）是试炼机制复杂度的核心来源。标签数量随层数递增：

| 层数区间 | 每只怪标签数 | 标签等级 | 说明 |
|---------|-------------|---------|------|
| 1-10    | 0-1 个      | Lv.1    | 偶尔出现单个基础标签 |
| 11-20   | 1 个        | Lv.1-2  | 每只怪固定 1 个标签 |
| 21-40   | 1-2 个      | Lv.2-3  | 开始标签组合 |
| 41-60   | 2 个        | Lv.3-4  | 双标签成为常态 |
| 61-80   | 2-3 个      | Lv.4-5  | 高等级标签叠加 |
| 80+     | 3 个        | Lv.4-5  | 满标签高等级 |

**标签选取规则**：
1. 优先从行为模板的 `defaultTags` 和 `optionalTags` 中选取
2. `optionalTags` 有引入章节限制（如 `packBonus` 从 Ch2 起），试炼中映射为层数：`引入章节 × 5 = 引入层数`
3. 同一层内避免冲突标签（如 `lifesteal` + `deathExplode` 在同一只怪上意义不大）

**组合示例**：

| 层数 | 行为 | 标签组合 | 玩家体验 |
|------|------|---------|---------|
| 5    | swarm | 无标签 | 纯数值压力，刷起来轻松 |
| 15   | bruiser | `slowOnHit Lv.2` | 被减速，需要走位 |
| 25   | tank + glass 混搭 | tank: `hpRegen Lv.2`; glass: `defPierce Lv.2` | 坦克回血拖时间，玻璃穿甲打输出 |
| 40   | debuffer + caster | `venomStack Lv.3` + `healAura Lv.2` | 毒叠层削血，治疗光环回复队友 |
| 65   | bruiser + tank + glass | `lifesteal Lv.3` + `sporeCloud Lv.3` + `firstStrikeMul Lv.3` | 多重机制叠加，需要极强 build |
| 90   | 全 tank | `hpRegen Lv.3` + `healAura Lv.5` + `corrosion Lv.1` | 极限坦克墙，DPS 不够就推不动 |

## §4 经验掉落

### 4.1 现状问题

当前试炼经验使用 `scaleMul^0.3` 作为经验倍率：

```lua
-- 当前公式（EndlessTrial.lua）
expScaleMul = scaleMul ^ 0.3
```

| scaleMul | expScaleMul (`^0.3`) | 实际经验倍率 |
|----------|---------------------|-------------|
| 2.0      | 1.23                | 仅 1.23× |
| 10.0     | 2.00                | 仅 2× |
| 100.0    | 3.98                | 仅 4× |

惩罚过重，试炼经验极低，与"高难度高收益"定位矛盾。

### 4.2 结算方式：逐层通关一次性发放

**每层通关时一次性发放该层经验，每层只能领取一次**：

- **通关某层** → 立即获得该层全部经验
- **死在某层** → 该层不给经验，可重试
- **已通关的层** → 永远不可重复领取

玩家从试炼获得的**终身总经验** = 所有已通关层的经验之和。

**设计意图**：
- 不可重复 → 不存在"刷低层赚经验"的 farming
- 通关即发 → 即时正反馈，推过一层就落袋为安
- 死亡无惩罚（只是不给当层经验）→ 鼓励挑战，不怕试错

### 4.3 新公式：取消惩罚 + 试炼倍率

**每层经验**（通关时一次性发放）：

```
floorExp = normalCount(f) × avgExpDrop × floorScaleMul × trialExpMul
```

| 参数 | 值 | 说明 |
|------|---|------|
| `floorScaleMul` | 当层的 scaleMul | 与怪物难度同步增长 |
| `trialExpMul` | 3.0 | 试炼经验额外乘数，定位为"高经验模式" |

其中 `avgExpDrop` 为该层怪物行为模板的加权平均经验基准值：

| 行为模板 | expDrop 基准 |
|---------|-------------|
| swarm   | 8           |
| tank    | 20          |
| glass   | 12          |
| bruiser | 15          |
| debuffer| 12          |
| caster  | 18          |
| exploder| 10          |

### 4.3 每层总经验

每层怪物数量（沿用现有公式，微调上限）：

```
normalCount = min(8 + floor(floor/5) × 2, 40)
```

| 层数 | 怪物数 | 说明 |
|------|-------|------|
| 1    | 8     | 基础 |
| 10   | 12    | 渐增 |
| 20   | 16    | |
| 50   | 28    | |
| 80   | 40    | 上限 |

Boss 层（每 10 层）：1 只 Boss，经验 = 该层全部普通怪经验之和（相当于一层的双倍收益）。

### 4.4 经验收益示例

以 Ch6 玩家（`baseScaleMul ≈ 40万`）为例，假设平均 `template.expDrop = 12`（加权平均），`trialExpMul = 3.0`：

| 层数 | floorMul | floorScaleMul | 单只经验 | 怪物数 | 层总经验 |
|------|----------|--------------|---------|-------|---------|
| 1    | 1.0      | 40万          | 1440万   | 8     | 1.15亿  |
| 10   | 1.69     | 67.6万        | 2434万   | 12    | 2.92亿  |
| 20   | 3.03     | 121万         | 4356万   | 16    | 6.97亿  |
| 30   | 5.42     | 217万         | 7812万   | 20    | 15.6亿  |
| 50   | 17.3     | 692万         | 2.49亿   | 28    | 69.7亿  |

> **注**: 具体数值需要结合经验公式（`floor(300 × 1.04^lv)`）和玩家当前等级校准。以上数值展示的是量级关系——试炼高层的单层经验远超普通关卡，但玩家需要足够的 build 才能推到那个层数。

### 4.5 自然平衡机制

**为什么不需要人为限制经验**：

1. **难度即门槛**: 每层 +6% 的指数增长天然限制玩家推进深度
2. **一次性不可重复**: 每层经验只能领取一次，不存在重复 farming
3. **build 绑定**: 同一 build 的推进上限基本固定，卡在某层就必须回去提升 build 再来
4. **边际递减**: 虽然单层经验随层数增长，但通关时间也在增长（怪物更肉、机制更复杂）
5. **trialExpMul 可调**: 如果经验过高/过低，只需调整一个参数即可全局校准

## §5 Boss 层设计

### 5.1 Boss 选取

每 10 层出现 Boss，沿用当前 Boss 池循环（14 个 Boss）。Boss 不使用 `MonsterTemplates.Assemble()`，保持手工设计的独特性。

```
Boss 层: floor % 10 == 0
Boss 选取: BOSS_MONSTERS[(floor/10 - 1) % 14 + 1]
Boss scaleMul: floorScaleMul × 2.0
Boss 限时: max(30, 60 - floor)  -- 高层限时更紧
```

### 5.2 Boss 经验

Boss 经验 = 该层等价普通怪总经验：

```lua
bossExpDrop = normalCount(floor) × avgExpDrop × expScaleMul
```

设计意图：Boss 层是"高风险高回报"的关键节点，经验丰厚但限时紧张。

## §6 抗性轮换

**保留轮换机制，替换抗性来源**。

| 维度 | 当前设计 | 新设计 |
|------|---------|-------|
| 轮换机制 | 每 5 层切换一组 | **保留**，周期可配置（`RESIST_ROTATION_PERIOD`，默认 5） |
| 抗性来源 | 独立 `RESIST_ROTATION` 表（6 组硬编码值） | `MonsterTemplates.Resists`（12 种模板） |
| 抗性格式 | 直接覆盖怪物抗性（`resistOverride`） | 作为 `resistId` 传入 `Assemble()`，由模板系统计算最终值 |
| 轮换深度 | 6 组 × 5 层 = 30 层一轮 | 12 种 × 5 层 = 60 层一轮 |

**好处**：
- 抗性值与怪物模板系统统一维护，改一处全局生效
- 12 种模板比 6 组覆盖更丰富，轮换周期更长
- resistCoeff 按层数区间递增（§3.4），低层轮换是"风味差异"，高层轮换是"硬门槛"
- 玩家仍可记住固定序列来预判，保留策略深度

详细轮换序列和 resistCoeff 分段见 §3.4。

## §7 实现要点

### 7.1 代码改动范围

| 文件 | 改动 |
|------|------|
| `EndlessTrial.lua` | 重写 `GetScaleMul()`（锚定玩家章节）、`BuildTrialQueue()`（接入 MonsterTemplates）、移除硬编码怪物列表 |
| `Spawner.lua` | 无需改动（已支持 `entry.expScaleMul`） |
| `MonsterTemplates.lua` | 无需改动（`Assemble()` 已输出 Spawner 兼容格式） |

### 7.2 关键数据流

```
EndlessTrial.BuildTrialQueue(floor)
  ├─ playerChapter = GameState.chapter
  ├─ baseScaleMul = StageConfig.CalcScaleMul(playerChapter, playerStage)
  ├─ floorScaleMul = baseScaleMul × 1.06^(floor-1)
  ├─ behaviorId = 按层数策略选取
  ├─ resistId = 按轮换序列选取（每 RESIST_ROTATION_PERIOD 层切换）
  ├─ tags = 按层数策略叠加
  ├─ template = MonsterTemplates.Assemble(behaviorId, resistId, chapter, {tags})
  └─ return { templateId, template, scaleMul }
        ↓
  Spawner.SpawnEnemy(entry)
    ├─ hp = template.hp × scaleMul
    ├─ atk = template.atk × scaleMul^0.75
    ├─ def = template.def × scaleMul
    └─ resist = template.resist（MonsterTemplates 已计算好）
        ↓
  通关当层时:
    floorExp = normalCount × avgExpDrop × floorScaleMul × trialExpMul
    → 一次性发放，标记该层已领取
```

### 7.3 配置参数汇总

| 参数 | 默认值 | 说明 |
|------|-------|------|
| `FLOOR_GROWTH_RATE` | 0.06 | 每层 scaleMul 增长率 |
| `BOSS_FLOOR_MUL` | 2.0 | Boss 层额外倍率 |
| `TRIAL_EXP_MUL` | 3.0 | 试炼经验额外乘数 |
| `NORMAL_COUNT_BASE` | 8 | 基础怪物数量 |
| `NORMAL_COUNT_INCR` | 2 | 每 5 层增加数量 |
| `NORMAL_COUNT_MAX` | 40 | 怪物数量上限 |
| `RESIST_ROTATION_PERIOD` | 5 | 抗性轮换周期（每 N 层切换一组 resistId） |
| `BOSS_TIMER_BASE` | 60 | Boss 限时基础（秒） |
| `BOSS_TIMER_MIN` | 30 | Boss 限时下限（秒） |
