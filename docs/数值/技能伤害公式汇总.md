> 版本: v3  
> 日期: 2026-03-15  
> 对应代码: `scripts/Config.lua`(SKILLS), `scripts/SkillTreeConfig.lua`, `scripts/battle/DamageFormula.lua`

## 统一伤害管线（六区乘算）

所有伤害（普攻 + 技能）共用同一管线：

```
单次伤害 = 基础区 × 增伤区 × 双爆区 × 减防区 × 抗性区 × 反应区   (六区管线, 普攻/技能共用)

区1 基础区 = totalAtk × multiplier
区2 增伤区 = 1 + sum(元素增伤, 药水, 印记, 套装, 天赋extraBonuses...)
区3 双爆区 = 暴击时 critDmg (如1.8), 否则 1.0
区4 减防区 = 1 - enemyDef / (enemyDef + K)    K随章节缩放
区5 抗性区 = 三段曲线 (见下方)
区6 反应区 = reactionDmgBonus (触发反应时, 否则 1.0)

管线外: + 沧溟之怒固定伤害
最终: max(1, floor(dmg))
```

### DPS 缩放结构（管线外乘区）

六区管线计算的是**单次伤害**，实际 DPS 还取决于管线外的频率/乘区：

```
普攻 DPS = 单次伤害 × 攻速 × 弹体乘数
技能 DPS = 单次伤害 × (1 + 技能伤害) × (1 / 有效CD)
```

| 维度 | 普攻流 | 技能流 | 对称关系 |
|------|--------|--------|---------|
| 属性加点 | SPD → 攻速 | **精通** → CDR + 技能伤害 | 同一位置，互斥选择 |
| 装备乘区 | 攻速 `spd`（递减） | **技能伤害** `skillDmg`（独立乘区） | 竞争同一装备预算 |
| 天赋 | 弹体数(分裂/散射/连射) | 技能等级倍率 | 各自天赋投入 |
| 频率 | 攻速（递减无硬顶） | CDR（递减无硬顶） | 结构一致 |

### 属性点·精通（新增第 10 属性）

新增 **精通** 属性点（第 10 个），专属技能流。现有 9 个属性（含 luck）全部保留不动。

```
属性点·精通:
  → CDR 贡献: 每点提供 cdrPerPoint 的 CDR 原始值，汇入 CDR 递减公式
  → 技能伤害贡献: 每点提供 skillDmgPerPoint 的技能伤害加成

UI 显示: "精通  15pt"  (不展示派生数值, 和体力/韧性一致)
```

### 装备词缀·技能伤害 `skillDmg`

独立乘区，和攻速对称。**不在基础区加算**，避免被技能等级倍率稀释。

```
技能 DPS = 单次伤害 × (1 + skillDmg) × (1 / 有效CD)
                       └── 装备乘区 ──┘

skillDmg 来源（加算后整体乘算）:
  ├── 装备词缀 skillDmg
  ├── 套装被动 skillDmg
  └── 属性点·精通贡献

示例: 装备 skillDmg=0.25, 套装=0.10, 精通贡献=0.15
→ (1 + 0.50) = 1.50 → 技能伤害 ×1.5
```

### 冷却时间计算（CDR 递减公式）

CDR 改为**渐近线递减**，和攻速递减结构一致，取消硬顶：

```
cdMul = 1 - maxCDR × totalCDR / (totalCDR + K)

参数: maxCDR = 0.75 (渐近线, 最多减75%CD)
      K      = 0.50 (半值常数)
```

**CDR 来源（加算后统一进递减）**:

| 来源 | 数值 |
|------|------|
| 天赋·法力亲和 `mana_affinity` | lv × 4%（最多5级=20%） |
| 装备词缀 `skillCdReduce` | 副属性 |
| 套装被动 `skillCdReduce` | 如符文编织2件=15% |
| 属性点·精通 | 每点 cdrPerPoint |

**投入 → 实际 CDR 对照**:

| totalCDR | cdMul | 实际CDR | 说明 |
|----------|-------|---------|------|
| 0.20 | 0.786 | 21.4% | 仅天赋 |
| 0.35 | 0.691 | 30.9% | + 套装 |
| 0.50 | 0.625 | 37.5% | + 装备（≈旧硬顶水平） |
| 0.80 | 0.538 | 46.2% | + 精通属性点 |
| 1.20 | 0.471 | 52.9% | 重度投入 |
| 2.00 | 0.400 | 60.0% | 极限堆叠 |
| ∞ | 0.250 | 75.0% | 渐近线，不可达 |

**额外 CD 加速**（独立于 cdMul）:
- 符文编织6件：共鸣期间 cdFlowMul=2（CD 流速翻倍）
- 符文编织4件：触发时所有技能 CD -3s（扣固定值）

### 区4 减防区

```
减防系数 = 1 - effectiveDef / (effectiveDef + K)
K = ENEMY_DEF_K × scaleMul    -- K随章节缩放, 保持减免比例稳定
```

破甲技能(def_break)生效时: `effectiveDef = enemyDef × (1 - defReduceRate)`

### 区5 抗性区（三段曲线）

| 区间 | 公式 | 说明 |
|------|------|------|
| resist < 0 | `1 - resist / 2` | 负抗有收益但减半, 防止减抗叠太多 |
| 0 ≤ resist < 0.75 | `1 - resist` | 线性, 和之前一致 |
| resist ≥ 0.75 | `1 / (1 + resist / 4)` | 渐近线, 永远打不到0伤害 |

元素削弱(elem_weaken)生效时: `effectiveResist = baseResist - weakenRate`

### 区6 反应区

```
反应系数 = 1 + bonus
未触发反应时 = 1.0
```

**bonus 来源（加算）**：

| 来源 | 效果 | 备注 |
|------|------|------|
| 天赋·反应精通 `reaction_mastery` | +lv×5% | 通用，最多5级=25% |
| 天赋·元素过载 `elem_overload` | +3/6/10% | 连续反应增伤 |
| 天赋·烈焰链锁 `fire_reaction_boost` | +lv×15% | 仅火系反应 |
| 天赋·极寒共振 `ice_reaction_boost` | +lv×15% | 仅冰系反应 |
| 天赋·侵蚀深化 `poison_reaction_boost` | +lv×15% | 仅毒系反应 |
| 天赋·激流增幅 `water_reaction_boost` | +lv×15% | 仅水系反应 |
| 天赋·奥术催化 `arcane_reaction_boost` | +lv×15% | 仅奥术反应 |
| 装备词缀 `reactionDmg` | 装备数值 | 副属性 |
| 套装被动 `reactionDmg` | 套装数值 | 如灰烬之焰2件 |
| 套装BUFF·灰烬之焰4件 | +30% | 反应后5秒内 |
| 技能BUFF·元素狂热 `elem_frenzy` | 运行时数值 | 释放后持续6秒 |

反应区独立乘算，不和增伤区稀释，使元素反应流派天赋价值稳定。

> ⚠️ **代码现状差异**: 五元素专属天赋（烈焰链锁等）目前在 `ElementReactions.lua` 中作为独立额外伤害扣血，尚未并入区6乘区。实现六区管线时需迁移。

## 四、普攻体系公式

### 普攻基础命中

```
dmgMul = 1.0 + normal_enhance_lv × 0.10
走五区管线: multiplier=dmgMul, damageTag="normal", element="weapon"
```

### 分裂弹

```
baseDmg = totalAtk × 0.40 (SPLIT_DMG_RATIO)
        × (1 + bullet_mastery_lv × 0.06)    -- 旧线弹体精通
        × (1 + split_dmg_lv × 0.12)          -- 新线弹体强化

弹数 = (2 + shatter_lv) + split_count_lv × 2
反弹: 每次伤害 × (1 + ricochet_amp_lv × 0.12), 默认衰减10%
棱镜: 每次穿透/反弹 prismStacks+1 → +prismStacks × prism_lv × 12% (增伤区)
万华镜: 无限反弹, 存续(3+lv)s
```

### 散射

```
散射伤害 = baseDmgMul × 0.7 (普攻的70%)
散射弹数 = (2 + scatter_lv) + scatter_count_lv
穿透伤害 = 再 × 0.8, 穿透数 = scatter_pierce_lv
```

### 穿刺箭矢

```
穿透伤害 = baseDmgMul × 0.8 (普攻的80%)
穿透数 = normal_pierce_lv
额外暴击率 = normal_crit_pierce_lv × 5%
```

### 元素乱流（独立计算，不走五区管线）

```
chaosDmg = floor(totalAtk × chaosLv × 0.03 × (1 + chaos_intensity_lv × 0.15))
附着随机元素, 乱流共鸣: 附着时长 +0.5s/lv
```

## 五、缩放风险标注

当前所有技能 maxLevel 为 3~5，线性缩放在此范围内可控。以下列出**若未来开放更高等级需注意的风险**：

| 技能 | 当前公式 | maxLv | lv_max 倍率 | 风险 |
|------|---------|-------|------------|------|
| heat_wave | `2.0 + lv × 1.0` | 5 | 700% | 每级+100%ATK, 增速最猛 |
| fire_storm | `2.0 + lv × 0.8` | 5 | 600% | 每级+80%ATK |
| ice_barrage | `(0.6+lv×0.15)×(4+lv)` | 5 | 1215% | 倍率×波数双增长, 总伤爆炸 |
| poison_rain | `1.5 + lv × 0.4` | 5 | 350% | 每级+40%ATK |
| range/radius | 多个技能线性增长 | 3~5 | — | 范围无上限 |

**建议**: 若后续开放更高等级, 考虑改为递减公式: `base × (1 + bonus × lv / (lv + k))` 或设硬上限 `min(lv, cap)`。
