## 系统概述

参考暗黑4宝石系统，同一颗宝石镶嵌在不同装备类型上提供不同属性加成。

- **7 种宝石**: 红宝石、蓝宝石、绿宝石、黄宝石、紫水晶、钻石、骷髅
- **5 个品质**: 碎裂、普通、完美、皇家、宏伟
- **3 类装备**: 武器、防具（手套/靴子）、首饰（护符/戒指/项链）
- **仅橙装有孔**: 孔位数量随机

### 装备类型分类

| 类型 | 包含槽位 | 说明 |
|------|---------|------|
| 武器 | weapon | 攻击向加成 |
| 防具 | gloves, boots | 生存向加成 |
| 首饰 | amulet, ring, necklace | 抗性/辅助加成 |

## 宝石类型与效果

| 宝石 | 武器效果 | 防具效果 | 首饰效果 |
|------|---------|---------|---------|
| 红宝石 | +攻击力 (atk) | +生命值 (hp) | +火焰抗性 (fireRes) |
| 蓝宝石 | +暴击率 (crit) | +防御力 (def) | +冰霜抗性 (iceRes) |
| 绿宝石 | +暴击伤害 (critDmg) | +幸运 (luck) | +毒素抗性 (poisonRes) |
| 黄宝石 | +技能伤害 (skillDmg) | +攻速 (spd) | +奥术抗性 (arcaneRes) |
| 紫水晶 | +反应增伤 (reactionDmg) | +护盾比例 (shldPct) | +流水抗性 (waterRes) |
| 钻石 | +全元素增伤 (elemDmg) | +生命百分比 (hpPct) | +全元素抗性 (allRes) |
| 骷髅 | +生命偷取 (lifeSteal) | +生命回复 (hpRegen) | +防御力 (def) |

### 钻石特殊处理

- **武器 (全元素增伤)**: 使用 `elemDmg` 属性，base = 0.0025（等效于单元素增伤 base 0.0025，因为 elemDmg 同时加成 5 种元素）
- **首饰 (全元素抗性)**: 5 种抗性各加 `base × 0.4 × gemMul × tierMul`（打 4 折，防止 5 抗叠加过强）

|------|--------|--------|---------|---------|
| 1 | 碎裂 | 0.15 | 20% | - |

## 装备孔位

**仅橙色品质装备拥有孔位**，上限 3 孔。

### 初始孔位

装备生成时随机决定初始孔数：

| 孔数 | 概率 |
|------|------|
| 0 孔 | 50% |
| 1 孔 | 39% |
| 2 孔 | 10% |
| 3 孔 | 1% |

- 期望初始孔数 = 0×0.50 + 1×0.39 + 2×0.10 + 3×0.01 = **0.62 孔/件**

### 打孔

所有橙装（含 0 孔）均可通过消耗 **散光棱镜** 打孔，每次增加 1 孔，上限 3 孔。

| 操作 | 散光棱镜消耗 |
|------|-------------|
| 0→1 孔 | 1 |
| 1→2 孔 | 2 |
| 2→3 孔 | 4 |

- 单件打满 3 孔最多需要 7 颗散光棱镜（从 0 孔起）
- 6 件全打满需要最多 42 颗散光棱镜
- 散光棱镜产出方式待定

### UI 表现

装备详情面板头部右侧空白区域（与名称/战力/强化三行等高），横向排列宝石孔位方块：

- **已镶嵌孔位**: 显示宝石颜色图标，点击弹出详情/拆卸
- **空孔位**: 灰色虚框，点击弹出宝石选择列表
- **可打孔**: 显示 "+打孔" 按钮，点击消耗散光棱镜增加 1 孔
- **已满 3 孔**: 不显示打孔按钮

## 数值公式

### 基础公式

```
宝石属性值 = base × gemMul × tierMul(装备所在章节)
```

- `base`: 复用 `Config.EQUIP_STATS` 中对应属性的基础值
- `gemMul`: 宝石品质系数（见上表）
- `tierMul`: 装备所在章节的 Tier 缩放系数（与装备词条共用同一公式）

### 定价锚点

宏伟宝石（gemMul=0.75）对标橙装副词条中值（base × 0.9 × tierMul）：

```
宏伟宝石 / 橙副词条中值 = 0.75 / 0.9 ≈ 83%
```

即 1 颗宏伟宝石 ≈ 0.83 条橙装副词条。

### 满镶嵌战力估算

3 孔满宏伟 = 0.75 × 3 = 2.25 倍 base × tierMul ≈ **2.5 条橙副词条** ≈ 多半件橙装的副词条量。

## 数值校验

以 ch6（tierMul ≈ 39.5）为例，验证宏伟宝石的实际数值是否合理。

### 宏伟宝石 ch6 实际值 vs 橙副词条中值

| 宝石 | 效果 | 宏伟实际值 | 橙副词条中值 | 比例 |
|------|------|-----------|------------|------|
| 红宝石·武器 | +atk | 1.50×39.5 = **59.3** | 2.0×0.9×39.5 = 71.1 | 83% |
| 红宝石·防具 | +hp | 24.0×39.5 = **948** | 32×0.9×39.5 = 1138 | 83% |
| 蓝宝石·武器 | +crit | 0.15%×39.5 = **5.9%** | 0.2%×0.9×39.5 = 7.1% | 83% |
| 蓝宝石·防具 | +def | 0.975×39.5 = **38.5** | 1.3×0.9×39.5 = 46.2 | 83% |
| 绿宝石·武器 | +critDmg | 0.30%×39.5 = **11.9%** | 0.4%×0.9×39.5 = 14.2% | 83% |

比例恒定 83%（0.75/0.9），符合设计预期。

### 满配战力增量估算（ch6，6件全橙，期望3.7孔全宏伟）

假设 3.7 颗宏伟红宝石全镶武器（最大化攻击力）：

```
+atk = 59.3 × 3.7 ≈ 219
```

对比 ch6 玩家 ATK（约 1500~2500），增量约 9%~15%，有感但不失衡。

### 极端情况：3孔武器全宏伟红宝石

```
+atk = 59.3 × 3 = 177.9
```

约等于 2.5 条橙副词条，相当于多穿半件橙装的副词条贡献。
