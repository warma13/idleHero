## 装备产出触发

装备产出有三个来源，每个来源的品质控制方式不同：

### 1. 怪物击杀掉落（随机品质）

每个怪物死亡时，按其掉落模板中的 `equipChance` 概率决定是否掉装备。幸运值会加成这个概率：

```
实际掉率 = equipChance + luck × 0.5
```

掉落模板决定基础概率：

| 模板 | equipChance | goldChance | 金币范围 |
|------|-------------|------------|---------|
| common（小怪） | 12% | 30% | 2~7 |
| elite（精英） | 20% | 50% | 5~12 |
| miniboss（小Boss） | 50% | 100% | 8~18 |
| boss（大Boss） | 100% | 100% | 20~40 |
| summon（召唤物） | 0% | 10% | 0~1 |

金币范围会再乘以 `√scaleMul` 做关卡缩放。

### 2. 保底掉落（指定最低品质）

关卡结算时，某些场景会调用 `CreateEquip(minQuality, chapter)` 直接生成指定品质的装备，跳过品质权重抽取。

### 3. 锻造系统（固定橙色）

锻造固定产出橙色品质，套装100%来自对应分段的套装池。详见下方锻造章节。

## 品质抽取

装备有5个品质等级，通过加权随机抽取：

| 品质 | 基础权重 | qualityMul | 副词条数 | 可套装 | 可升级 |
|------|---------|------------|---------|--------|--------|
| 白色 | 50 | 1.0 | 0 | 否 | 否 |
| 绿色 | 30 | 1.5 | 1 | 否 | 10级 |
| 蓝色 | 15 | 2.0 | 2 | 是 | 20级 |
| 紫色 | 4 | 3.0 | 3 | 是 | 35级 |
| 橙色 | 1 | 5.0 | 4 | 是 | 50级 |

### 权重调整

抽取时有两个修正：

**幸运值修正**：白色和绿色的权重被压低（`× max(0.3, 1-luck)`），蓝色及以上的权重被提升（`× (1 + luck × (品质等级-2))`）。幸运值越高，出高品质的概率越大。

**非Boss压权**：小怪掉落时，蓝色及以上品质被大幅压低。具体倍率为 `{1, 1, 0.054, 0.02, 0.008}`，也就是小怪出蓝色的权重只有Boss的5.4%，紫色2%，橙色0.8%。这意味着小怪出橙色几乎不可能，橙色装备主要来源是Boss和锻造。

## 槽位决定

从6个装备槽位中等概率随机一个：武器、手套、护符、戒指、靴子、项链。

锻造模式可以花双倍消耗锁定指定槽位。

## 主词条生成

每个槽位有一个 `mainPool`（2~3个候选词条），从中加权随机选一个。

### 加权规则

被标记为 `isAtkStat`（攻击系词条：atk、spd、crit、critDmg、五种单元素增伤）的词条权重为 **1**，其他词条权重为 **2**。这意味着非攻击系词条出现概率是攻击系的两倍，目的是让攻击力类词条更稀缺、更有价值。

### 各槽位主词条池

| 槽位 | 候选池 | 说明 |
|------|--------|------|
| 武器 | atk, reactionDmg, fireDmg, iceDmg, poisonDmg, arcaneDmg, waterDmg | atk/五元素 权重1, reactionDmg 权重2 |
| 手套 | spd, crit, skillDmg | spd/crit 权重1, skillDmg 权重2 |
| 护符 | crit, hpPct, critDmg, fireDmg, iceDmg, poisonDmg, arcaneDmg, waterDmg | crit/critDmg/五元素 权重1, hpPct 权重2 |
| 戒指 | critDmg, reactionDmg, atk | critDmg/atk 权重1, reactionDmg 权重2 |
| 靴子 | def, hpPct, spd | spd 权重1, 其余 权重2 |
| 项链 | luck, skillDmg, reactionDmg | 全部 权重2 |

### 主词条数值

```
主词条值 = base × mainMul × qualityMul × tierMul
```

- `base`：该词条的基础值（Config.EQUIP_STATS 中定义）
- `mainMul`：主词条相对副词条的倍率（统一为 20）
- `qualityMul`：品质系数（白1.0, 绿1.5, 蓝2.0, 紫3.0, 橙5.0）
- `tierMul`：章节缩放（所有属性统一）

---

## 副词条生成

### 数量

由品质决定：白0条、绿1条、蓝2条、紫3条、橙4条。

### 选取方式

从槽位的 `subPool`（7~9个候选）中 Fisher-Yates 洗牌，取前 N 个。**不会与主词条重复**（因为 mainPool 和 subPool 本身是分开定义的，有少量重叠但概率自然分散）。

### 副词条数值

```
副词条值 = base × random(0.8, 1.0) × tierMul
```

- `base`：该词条的基础值
- `random(0.8, 1.0)`：浮点随机，让同一词条不同装备有微小差异
- `tierMul`：章节缩放（所有属性统一）

### 各槽位副词条池

| 槽位 | 副词条候选 |
|------|-----------|
| 武器 | spd, crit, critDmg, hp, lifeSteal, luck, skillDmg, fireRes, iceRes, fireDmg, iceDmg, poisonDmg, arcaneDmg, waterDmg |
| 手套 | atk, critDmg, reactionDmg, def, hpRegen, luck, skillDmg, hpPct, poisonRes, fireDmg, iceDmg, poisonDmg, arcaneDmg, waterDmg |
| 护符 | atk, spd, critDmg, def, shldPct, luck, fireRes, iceRes, fireDmg, iceDmg, poisonDmg, arcaneDmg, waterDmg |
| 戒指 | spd, crit, hp, hpRegen, shldPct, hpPct, skillDmg, waterRes, fireDmg, iceDmg, poisonDmg, arcaneDmg, waterDmg |
| 靴子 | atk, crit, critDmg, hpRegen, lifeSteal, fireRes, waterRes, fireDmg, iceDmg, poisonDmg, arcaneDmg, waterDmg |
| 项链 | atk, spd, crit, hp, def, shldPct, hpPct, iceRes, poisonRes, fireDmg, iceDmg, poisonDmg, arcaneDmg, waterDmg |

## Tier缩放（统一缩放）

所有属性（无论绝对值还是百分比）统一乘以 `tierMul`，无例外：

```
tierMul = 1 + 99 × ln(ch) / ln(100)
ch1=1.0, ch10≈50.5, ch100=100
```

- **主词条**: `base × mainMul × qualityMul × tierMul`
- **副词条**: `base × random(0.8, 1.0) × tierMul`

### 为什么百分比属性也缩放

百分比属性（crit、critDmg、hpPct 等）的 `base` 值本身很小（如 crit=0.002, critDmg=0.004），即使乘以 tierMul=100，副词条也只有 0.2/0.4，不会失控。同时：

- **暴击率**有 100% 硬顶，溢出部分转为暴伤（3:1 比例）
- **CDR**有渐近线递减机制
- **hpPct**等防御百分比受 STAT_IMPORTANCE 权重（0.6~0.7）自然压低

统一缩放的好处是：换高章节装备时**所有词条都有提升感**，不存在"百分比词条换装无意义"的问题。

## 套装标签

### 触发条件

蓝色品质及以上，50% 概率附带套装标签。

### 套装池筛选

- **Boss 掉落**：仅限当前章节批次的套装（严格匹配 `s.chapter == chapter`）
- **非 Boss 掉落**：所有已解锁章节的套装（`s.chapter <= chapter`）
- **已退役套装**（`retired = true`）不再掉落

从可选池中等概率随机一个。

## 装备升级

> **v4.1 重设计**：升级上限按品质递增（绿10/蓝20/紫35/橙50），升级不改变 IP，每5级随机一条词缀提升。

### 基本规则

- 白色不可升级，升级上限按品质递增：绿色10级、蓝色20级、紫色35级、橙色50级
- 升级**不改变 IP**，升级加成独立于 IP 体系

### 主属性成长（每级）

```
mainStatValue = slotBase × (IP/100) + slotBase × 升级等级 × 0.04
```

每级增加 slotBase × 4% 的固定值。橙色满级 50 时累计额外增加 2 倍 slotBase。

### 词缀里程碑（每5级）

每 5 级升级时，**随机选一条词缀**获得 +2% 乘算加成。每条词缀独立记录被选中次数（`milestoneCount`）。

```
词缀值 = baseValue × (1 + milestoneCount × 0.02)
```

橙色满级 50 时共 10 次里程碑机会，随机分配给各词缀。附魔替换词缀时 milestoneCount 归零。

### 升级消耗（强化石）

```
cost = max(2, floor(2 + lv×0.8 + lv²×0.03))
```

消耗随等级平缓增长。

## 魔法石Tier提升

蓝色及以上装备可以使用魔法石将装备 Tier 提升到更高章节：

### 缩放逻辑

```
scale = tierMul(目标章节) / tierMul(当前章节)
```

所有词条（主词条和副词条）统一乘以 `scale`，无例外。

这意味着一件低章节的好词条装备可以通过魔法石追上高章节的数值。

## 锻造系统

### 规则

- 每日 1 次免费 + 10 次付费，共 11 次
- 固定产出**橙色**品质
- 套装 100% 来自对应分段的套装池
- 可花双倍消耗锁定指定槽位
- 词条生成算法与普通掉落完全一致（主词条加权、副词条洗牌、统一tierMul缩放）

### 分段

每 4 章一段，Tier 取该段内玩家已通关的最高 Boss 关对应的章节：

| 段 | 章节范围 |
|----|---------|
| 1: 荒原·炎狱 | 1~4 章 |
| 2: 深海·雷霆 | 5~8 章 |
| 3: 圣域·深渊 | 9~12 章 |

### 消耗

| 类型 | 强化石 | 金币 |
|------|--------|------|
| 普通 | 160 | 30 × √scaleMul |
| 锁部位 | 320 | 60 × √scaleMul |
